🔗 Figma para Design de Interface · Parte 3

Prototipagem Interativa, Testes de Usabilidade e Projeto Final

Parabéns por chegar até aqui! Agora você vai aprender a transformar seus designs em protótipos funcionais, simular interações, testar com usuários e entregar um projeto completo. Ao final, um projeto guia consolida tudo o que você aprendeu nas três partes práticas.

1. Prototipagem: conectando telas com interações

No Figma, a prototipagem é feita conectando frames através de linhas de fluxo. Você define gatilhos (clique, arrasto, hover) e ações (navegar para tela, abrir overlay, scroll).

Modo Prototype

Clique na aba Prototype (canto superior direito, ao lado de Design). Selecione um elemento ou frame, arraste o círculo azul (⚪) até o destino.

Figura 1 – Conexão de protótipo: clique no botão leva para outra tela.

Configurações de interação

ConfiguraçãoOpçõesQuando usar
Gatilho (Trigger)On Click, On Drag, While Hover, After DelayClique é o mais comum. Hover para menus suspensos.
Ação (Action)Navigate to, Open Overlay, Scroll to, BackNavigate to = troca de tela. Overlay = modal, menu lateral.
AnimaçãoInstant, Dissolve, Push, Slide, Smart AnimateSmart Animate cria transições suaves entre elementos similares.
Smart Animate é um recurso incrível: se dois frames têm elementos com o mesmo nome, o Figma anima automaticamente a transição (posição, tamanho, cor, opacidade). Ótimo para microinterações!
Para testar seu protótipo, clique no botão Present (ícone de play) no canto superior direito. Você pode gerar um link compartilhável clicando em Share Prototype.
🔗 MICROATIVIDADE – Crie um fluxo de 3 telas Crie três frames: Tela Inicial → Tela de Detalhes → Tela de Confirmação. Adicione botões e conecte com interações "On Click → Navigate to". Teste o protótipo no modo Present.

2. Componentes interativos avançados

Nem toda interação precisa trocar de tela. Overlays (modais, pop-ups, menus) mantêm o contexto enquanto exibem conteúdo complementar.

Criando um overlay (modal)

  1. Desenhe o conteúdo do modal em um frame separado (ex: 300x200px).
  2. Selecione o botão que abrirá o modal → vá em Prototype.
  3. Conecte o botão ao frame do modal.
  4. Na configuração, escolha Action = Open Overlay.
  5. Defina posição (Manual, Center, Top, Bottom) e um fundo escurecido (Background overlay).
Figura 2 – Exemplo de modal aberto como overlay sobre a tela principal.

Scroll vertical e horizontal

Para simular listas roláveis (feed, produtos), selecione um frame e ative Overflow behavior = Vertical scrolling ou Horizontal scrolling no painel de Prototype.

Combine scrolling com Fixed elements (cabeçalho que não rola). Selecione um elemento no frame → Prototype → Scroll behavior → "Fix position when scrolling".

3. Testando protótipos com usuários reais

O Figma oferece recursos para validar seu design antes da programação. Além do modo Present, você pode usar plugins e integrações.

Ferramentas de teste

  • Figma Mirror (app mobile): visualize o protótipo no celular em tempo real.
  • Plugin Maze: converte seu protótipo em teste de usabilidade com métricas (taxa de sucesso, tempo, mapas de calor).
  • Compartilhar link com observação: ao gerar o link do protótipo, marque a opção "Password protect" ou "Allow viewers to leave comments".
Figura 3 – Sessão de teste de usabilidade com protótipo Figma.

Roteiro de teste prático

  1. Defina tarefas: ex: "Encontre um produto e adicione ao carrinho".
  2. Recrute 3-5 usuários: colegas ou pessoas do público-alvo.
  3. Observe e anote: onde hesitaram, onde clicaram errado, quanto tempo levaram.
  4. Itere: corrija os problemas no design e teste novamente.
O plugin Utterly permite gravar sessões de teste de usabilidade diretamente no Figma, com áudio e vídeo.
🧪 EXERCÍCIO – Teste com um colega Pegue o protótipo de 3 telas que você criou. Peça para um colega realizar a tarefa: "Clique no botão da tela inicial, depois no botão de confirmação na segunda tela". Observe se ele consegue sem ajuda. Documente o que aconteceu.

4. Trabalhando em equipe e recebendo feedback

O Figma é nativamente colaborativo. Várias pessoas podem editar o mesmo arquivo simultaneamente, como um Google Docs para design.

Compartilhando arquivos

  • Clique em Share (botão azul no topo direito).
  • Defina permissões: Can view (só visualizar), Can comment (deixar comentários), Can edit (editar).
  • Envie o link por e-mail ou copie para compartilhar.

Deixando e respondendo comentários

No modo View ou Edit, clique no ícone de balão de comentários (sidebar esquerda) ou pressione C. Clique em qualquer lugar do canvas, escreva seu feedback e marque colegas com @nome.

Figura 4 – Fluxo de revisão com comentários ancorados em elementos específicos.
Use o Modo de observação (ícone de olho no canto superior direito) para seguir o cursor de um colega durante uma reunião de design remota.

🎯 PROJETO FINAL – Aplicativo de Tarefas (To-Do List)

Este projeto consolida TUDO que você aprendeu nas 3 partes práticas. Siga os passos abaixo e entregue um protótipo funcional.

Etapa 1 – Estrutura e Design System (Partes 1 e 2)

  • Crie um arquivo Figma chamado "Projeto Final - To-Do List".
  • Crie uma página "Design System" com: 3 estilos de cor (Primária #A259FF, Secundária #2C7DA0, Cinza claro para fundos), 2 estilos de texto (Título e Corpo).
  • Crie componentes: Botão Primário, Botão Secundário, Card de Tarefa (com checkbox, título, data).

Etapa 2 – Telas e Layout (Partes 1 e 2)

  • Crie 3 frames (mobile 390x844): "Lista de Tarefas", "Adicionar Tarefa", "Detalhes da Tarefa".
  • Aplique grid de colunas (mobile: 4 colunas) nos frames.
  • Use Auto Layout para organizar listas e formulários.

Etapa 3 – Prototipagem (Parte 3)

  • Conecte as telas: botão "+" → navega para "Adicionar Tarefa"; botão "Salvar" → volta para "Lista de Tarefas".
  • Adicione um modal de confirmação ao excluir uma tarefa (overlay).
  • Teste o protótipo no modo Present.

Etapa 4 – Teste e Feedback (Parte 3)

  • Compartilhe o link do protótipo com 2 colegas. Peça que realizem as tarefas: "Adicionar uma nova tarefa" e "Marcar como concluída".
  • Colete os comentários diretamente no Figma.
  • Faça pelo menos uma iteração com base no feedback recebido.

✅ Entregável: Link do arquivo Figma com todas as telas, componentes e protótipo funcional. Você terá um portfólio sólido para mostrar em entrevistas!

5. Dicas finais: do Figma para o mundo real

Como preparar arquivos para desenvolvedores

  • Inspect mode: Desenvolvedores podem clicar em qualquer elemento e ver CSS, iOS, Android nativamente (altura, largura, cores, fontes).
  • Handoff com Zeplin ou Figma dev mode: o modo Dev (pago) facilita ainda mais a entrega.
  • Exporte assets organizados: ícones em SVG, imagens em 1x/2x/3x.

Versionamento de projetos

O Figma salva automaticamente o histórico de versões. Acesse File → Show version history. Você pode nomear versões importantes (ex: "V1 - Wireframes", "V2 - Protótipo testado").

Figura 5 – Histórico de versões: nunca perca seu trabalho.
Figma também suporta Branching e Merging (similar ao Git) para equipes grandes. Você pode criar um branch para experimentar mudanças sem afetar o arquivo principal.

📌 Resumo – Parte 3 (e conclusão da trilha prática)

  • Prototipagem: conecte frames com interações (On Click, Smart Animate, Overlays).
  • Testes de usabilidade: use Figma Mirror, Maze ou sessões presenciais para validar seu design.
  • Colaboração: compartilhe arquivos, use comentários e Modo de Observação.
  • Projeto Final: construa um aplicativo To-Do List completo com Design System, Auto Layout e protótipo testado.
  • Handoff: prepare assets e use o modo Inspect para devs.
