1. Objetos: agrupando dados com significado

Objetos são uma das estruturas de dados mais importantes em JavaScript. Eles armazenam pares de chave-valor (também chamados de propriedades), onde a chave é uma string (ou Symbol) e o valor pode ser qualquer tipo de dado: string, número, booleano, array, função, ou até outro objeto.

Representação visual de um objeto: chaves apontando para valores
// Criando um objeto literal (a forma mais comum)
const pessoa = {
    nome: "Carlos Silva",
    idade: 32,
    email: "carlos@email.com",
    endereco: {
        rua: "Av. Paulista",
        numero: 1000,
        cidade: "São Paulo"
    },
    // Método (função dentro do objeto)
    saudacao: function() {
        return `Olá, meu nome é ${this.nome} e tenho ${this.idade} anos.`;
    },
    // Sintaxe moderna de método (ES6)
    aniversario() {
        this.idade++;
    }
};

// Acessando propriedades (duas formas)
console.log(pessoa.nome);           // "Carlos Silva"
console.log(pessoa["idade"]);       // 32
console.log(pessoa.endereco.cidade); // "São Paulo"
console.log(pessoa.saudacao());      // "Olá, meu nome é Carlos Silva..."

// Adicionando novas propriedades
pessoa.telefone = "(11) 99999-8888";
pessoa["estaAtivo"] = true;

// Removendo propriedades
delete pessoa.email;

// Verificando se uma propriedade existe
console.log("nome" in pessoa);       // true
console.log(pessoa.hasOwnProperty("email")); // false (foi deletado)

1.1 Funções construtoras e a palavra-chave new

Quando precisamos criar vários objetos com a mesma estrutura, usamos funções construtoras (por convenção, com a primeira letra maiúscula). O operador new cria um novo objeto e o associa ao this da função.

// Função construtora
function Produto(nome, preco, estoque) {
    this.nome = nome;
    this.preco = preco;
    this.estoque = estoque;
    
    this.aplicarDesconto = function(percentual) {
        if (percentual >= 0 && percentual <= 100) {
            this.preco = this.preco * (1 - percentual / 100);
        }
    };
    
    this.valorEmEstoque = function() {
        return this.preco * this.estoque;
    };
}

// Criando instâncias
const notebook = new Produto("Notebook Dell", 3500, 10);
const mouse = new Produto("Mouse Logitech", 120, 50);

notebook.aplicarDesconto(15);
console.log(notebook.preco);              // 2975
console.log(notebook.valorEmEstoque());   // 29750
console.log(mouse.preco);                 // 120

1.2 Classes (ES6 – sintaxe moderna)

A partir do ECMAScript 2015 (ES6), JavaScript introduziu a sintaxe de classes, que é um açúcar sintático sobre as funções construtoras, mas torna o código mais organizado.

class Usuario {
    constructor(nome, email, idade) {
        this.nome = nome;
        this.email = email;
        this.idade = idade;
        this.dataCadastro = new Date();
    }
    
    saudacao() {
        return `Olá, eu sou ${this.nome}.`;
    }
    
    fazerAniversario() {
        this.idade++;
        console.log(`${this.nome} agora tem ${this.idade} anos!`);
    }
    
    // Método estático (chamado na classe, não na instância)
    static compararIdade(usuario1, usuario2) {
        return usuario1.idade - usuario2.idade;
    }
}

const user1 = new Usuario("Ana", "ana@email.com", 25);
const user2 = new Usuario("Bruno", "bruno@email.com", 30);

console.log(user1.saudacao());  // "Olá, eu sou Ana."
user2.fazerAniversario();       // "Bruno agora tem 31 anos!"

// Método estático
console.log(Usuario.compararIdade(user1, user2)); // -5 (Ana é mais nova)

1.3 Iterando sobre objetos: for...in e métodos modernos

const carro = { marca: "Toyota", modelo: "Corolla", ano: 2022, cor: "Prata" };

// for...in (percorre propriedades enumeráveis)
for (let chave in carro) {
    console.log(`${chave}: ${carro[chave]}`);
}

// Métodos modernos para trabalhar com objetos
const chaves = Object.keys(carro);     // ["marca", "modelo", "ano", "cor"]
const valores = Object.values(carro);  // ["Toyota", "Corolla", 2022, "Prata"]
const entradas = Object.entries(carro); // [["marca","Toyota"], ["modelo","Corolla"], ...]

// Usando for...of com entries
for (let [chave, valor] of Object.entries(carro)) {
    console.log(`${chave} = ${valor}`);
}

1.4 Arrays: listas ordenadas

Arrays são estruturas que armazenam valores indexados numericamente (começando do índice 0). Podem conter qualquer tipo de dado, inclusive outros arrays (matrizes) ou objetos.

// Criando arrays
const frutas = ["maçã", "banana", "laranja"];
const numeros = [1, 2, 3, 4, 5];
const misto = ["texto", 42, true, null, { nome: "objeto" }, [1, 2, 3]];

// Acessando elementos
console.log(frutas[0]);     // "maçã"
console.log(frutas.length); // 3

// Métodos de array fundamentais
frutas.push("uva");         // adiciona no final → ["maçã", "banana", "laranja", "uva"]
const ultima = frutas.pop(); // remove e retorna o último → "uva"
frutas.unshift("morango");  // adiciona no início → ["morango", "maçã", "banana", "laranja"]
const primeira = frutas.shift(); // remove e retorna o primeiro → "morango"

// Encontrar posição
console.log(frutas.indexOf("banana")); // 1
console.log(frutas.includes("laranja")); // true

// Fatiar array (sem modificar original)
const citricas = frutas.slice(1, 3);  // ["banana", "laranja"] (índice 1 até 2)

// Remover/substituir elementos com splice
frutas.splice(1, 1, "kiwi", "pera"); // remove 1 elemento no índice 1 e adiciona kiwi e pera

1.5 Iterando sobre arrays

const numeros = [10, 20, 30, 40, 50];

// 1. for tradicional (com índice)
for (let i = 0; i < numeros.length; i++) {
    console.log(`Índice ${i}: ${numeros[i]}`);
}

// 2. for...of (recomendado quando não precisa do índice)
for (let valor of numeros) {
    console.log(valor);
}

// 3. forEach (método funcional)
numeros.forEach(function(valor, indice) {
    console.log(`${indice}: ${valor}`);
});

// 4. map (cria um novo array transformado)
const dobros = numeros.map(n => n * 2);  // [20, 40, 60, 80, 100]

// 5. filter (filtra elementos)
const maioresQue30 = numeros.filter(n => n > 30); // [40, 50]

// 6. reduce (acumula valores)
const soma = numeros.reduce((acumulador, atual) => acumulador + atual, 0); // 150

2. Funções: reutilizando código

2.1 Declaração de função vs. Arrow function vs. Expressão de função

Tipo
SintaxeHoistingthisQuando usar
Declaração de função function soma(a,b) { return a+b; } Sim (pode ser chamada antes da declaração) Dinâmico (depende de como é chamada) Funções principais, métodos de objeto
Expressão de função const soma = function(a,b) { return a+b; }; Não (a variável tem hoisting, mas a atribuição não) Dinâmico Callbacks, funções anônimas
Arrow function (ES6+) const soma = (a,b) => a + b; Não Lexical (herda o this do escopo pai) Callbacks, funções curtas, quando precisa preservar o this 
// Exemplos práticos
// Declaração (pode ser chamada antes)
console.log(soma(5, 3)); // 8
function soma(a, b) {
    return a + b;
}

// Arrow function (sintaxe concisa)
const multiplicar = (a, b) => a * b;
const quadrado = x => x ** 2;  // um parâmetro: parênteses opcionais
const saudacao = () => "Olá, mundo!"; // sem parâmetros

// Arrow function com corpo (várias linhas)
const calcularImc = (peso, altura) => {
    const imc = peso / (altura ** 2);
    if (imc < 18.5) return "Abaixo do peso";
    if (imc < 25) return "Peso normal";
    if (imc < 30) return "Sobrepeso";
    return "Obesidade";
};

// Parâmetros padrão (default parameters)
function cumprimentar(nome = "Visitante", saudacao = "Olá") {
    return `${saudacao}, ${nome}!`;
}
console.log(cumprimentar());               // "Olá, Visitante!"
console.log(cumprimentar("João"));         // "Olá, João!"
console.log(cumprimentar("Maria", "Oi"));  // "Oi, Maria!"

// Rest parameters (...args) – número variável de argumentos
function somarTodos(...numeros) {
    return numeros.reduce((total, n) => total + n, 0);
}
console.log(somarTodos(1, 2, 3, 4, 5)); // 15

2.2 Trabalhando com datas (Date)

O objeto Date é usado para trabalhar com datas e horários. É importante lembrar que os meses vão de 0 (janeiro) a 11 (dezembro).

// Criando datas
const agora = new Date();                    // data e hora atual
const dataEspecifica = new Date(2026, 5, 15); // 15 de junho de 2026 (mês 5 = junho)
const timestamp = new Date(1700000000000);   // a partir de timestamp (milissegundos desde 1970)

// Métodos get
console.log(agora.getFullYear());   // ano com 4 dígitos
console.log(agora.getMonth());      // mês (0-11) → adicione +1 para humano
console.log(agora.getDate());       // dia do mês (1-31)
console.log(agora.getDay());        // dia da semana (0 = domingo, 6 = sábado)
console.log(agora.getHours());      // horas (0-23)
console.log(agora.getMinutes());    // minutos
console.log(agora.getSeconds());    // segundos

// Formatação para exibição
const opcoes = { weekday: 'long', year: 'numeric', month: 'long', day: 'numeric' };
console.log(agora.toLocaleDateString('pt-BR', opcoes)); // "terça-feira, 15 de junho de 2026"
console.log(agora.toLocaleTimeString('pt-BR'));         // "14:30:25"

// Calculando diferença entre datas
function diasEntreDatas(data1, data2) {
    const diffMs = Math.abs(data2 - data1);   // diferença em milissegundos
    return Math.ceil(diffMs / (1000 * 60 * 60 * 24)); // converte para dias
}

const inicio = new Date(2026, 0, 1);   // 1 de janeiro de 2026
const fim = new Date(2026, 11, 31);    // 31 de dezembro de 2026
console.log(`Dias no ano: ${diasEntreDatas(inicio, fim)}`);

3. BOM (Browser Object Model) – interagindo com o navegador

O BOM é um conjunto de objetos que permitem ao JavaScript interagir com a janela do navegador, fora do conteúdo da página em si. O objeto principal é o window (que é global no navegador).

Hierarquia do BOM: window → document, history, location, navigator, screen

3.1 Objeto window

// window é o objeto global (no navegador)
console.log(window.innerWidth);   // largura da janela de visualização (viewport)
console.log(window.innerHeight);  // altura da janela
console.log(window.outerWidth);   // largura da janela incluindo bordas
console.log(window.screenX);      // posição X da janela na tela

// window.open() – abre nova janela/aba
const novaJanela = window.open("https://www.google.com", "_blank");

// window.close() – fecha a janela (apenas se aberta por script)
// novaJanela.close();

// window.scrollTo() – rola a página
window.scrollTo({ top: 0, behavior: "smooth" });

console.log(navigator.userAgent);      // identificação do navegador
console.log(navigator.language);        // idioma do navegador ("pt-BR")
console.log(navigator.onLine);          // true/false – conectado à internet?

3.4 setTimeout e setInterval – temporizadores

// setTimeout – executa UMA VEZ após o atraso (em milissegundos)

    console.log("Isso aparece após 3 segundos");

// Cancelar o timeout antes de executar
// clearTimeout(timeoutId);

let contador = 0;
const intervalId = setInterval(() => {
    contador++;
    console.log(`Execução número ${contador}`);

function iniciarRelogio() {

        const horaFormatada = agora.toLocaleTimeString('pt-BR');
        console.log(horaFormatada);
        // Na prática, atualizaria um elemento HTML
    }, 1000);
}

4. DOM – manipulando a página dinamicamente

O DOM (Document Object Model) é a representação em árvore de todos os elementos HTML da página. Com ele, podemos acessar, modificar, adicionar e remover elementos dinamicamente.

Árvore DOM: document → html → head/body → elementos filhos

4.1 Selecionando elementos

Método
RetornaPerformanceExemplo
getElementById() Elemento único ou null Muito rápida document.getElementById("titulo")
querySelector() Primeiro elemento ou null Rápida document.querySelector(".classe")
document.querySelector("#id")
querySelectorAll() NodeList (estático, com forEach) Rápida document.querySelectorAll("p.card")
getElementsByClassName() HTMLCollection (viva, sem forEach) Muito rápida document.getElementsByClassName("item")
getElementsByTagName() HTMLCollection Muito rápida document.getElementsByTagName("div")

4.2 Modificando conteúdo, atributos e estilos

console.log(estilos.backgroundColor);

5. Eventos – respondendo às ações do usuário

Eventos são ações que ocorrem na página (cliques, teclas pressionadas, movimentos do mouse, carregamento, etc.). JavaScript pode "escutar" esses eventos e executar funções em resposta.

5.1 Principais eventos do DOM

Evento 77Página ou recurso termina de carregar
DescriçãoElementos típicos
click Usuário clica no elemento (mouse ou touch) Botões, links, qualquer elemento
dblclick Clique duplo Elementos que precisam de ação especial
mouseover / mouseout Mouse entra/sai do elemento Efeitos de hover, tooltips
focus / blur Campo recebe/perde foco Validação de formulário, placeholders
keydown / keyup Tecla pressionada/solta Jogos, atalhos de teclado, máscaras
submit Formulário é enviado Validação antes do envio, prevenção do comportamento padrão
load Inicializações seguras
scroll Usuário rola a página Efeitos de parallax, lazy loading, menu fixo

5.2 addEventListener vs. atributos HTML

<!-- Método 1: Atributo HTML (mais antigo, menos flexível) -->
<button onclick="alert('Clicou!')">Clique</button>
<button onclick="minhaFuncao()">Chamar função</button>

<!-- Método 2: addEventListener (RECOMENDADO) -->
<button id="meuBotao">Clique com addEventListener</button>

    console.log("Função chamada!");
}

    console.log("Botão clicado!");

    console.log("Clique tratado");
}
botao.addEventListener("click", handleClick);
// botao.removeEventListener("click", handleClick);
</script>

// window.load – espera TODOS os recursos (imagens, CSS, etc.)

6. Persistência com localStorage

O localStorage é uma API de armazenamento que permite salvar dados no navegador do usuário de forma persistente (os dados não expiram e permanecem mesmo após fechar o navegador). É ideal para salvar preferências, tarefas, carrinho de compras, etc.

Método
DescriçãoExemplo
setItem(chave, valor) Salva um par chave-valor (valor deve ser string) localStorage.setItem("nome", "João");
getItem(chave) Recupera o valor associado à chave (ou null) const nome = localStorage.getItem("nome");
removeItem(chave) Remove o item com a chave especificada localStorage.removeItem("nome");
clear() Remove TODOS os itens do localStorage localStorage.clear();
length Número de itens armazenados console.log(localStorage.length); 
// Salvando dados (valores precisam ser strings)
localStorage.setItem("usuario", JSON.stringify({ nome: "Ana", idade: 28 }));

// Recuperando dados
const usuarioString = localStorage.getItem("usuario");
if (usuarioString) {
    const usuario = JSON.parse(usuarioString);
    console.log(usuario.nome);  // "Ana"
}

// Exemplo prático: salvar e carregar tarefas
function salvarTarefas(tarefas) {
    localStorage.setItem("tarefas", JSON.stringify(tarefas));
}

function carregarTarefas() {
    const tarefasString = localStorage.getItem("tarefas");
    if (tarefasString) {
        return JSON.parse(tarefasString);
    }
    return [];  // retorna array vazio se não houver tarefas salvas
}

// Uso
let minhasTarefas = ["Estudar JS", "Fazer exercícios"];
salvarTarefas(minhasTarefas);

const tarefasCarregadas = carregarTarefas();
console.log(tarefasCarregadas);  // ["Estudar JS", "Fazer exercícios"]
⚠️ Limitações do localStorage:
- Capacidade: geralmente 5-10 MB por domínio.
- Armazena apenas strings (use JSON.stringify() e JSON.parse() para objetos/arrays).
- Síncrono: operações bloqueiam a thread principal (não use para grandes volumes de dados).
- Disponível apenas no lado do cliente (não é enviado ao servidor).

7. Projeto final: To-Do List completa com localStorage

🎯 PROJETO FINAL – Lista de Tarefas com Persistência

Desafio completo: Construa um aplicativo de lista de tarefas (To-Do List) com as seguintes funcionalidades:

📝 Requisitos obrigatórios:
  • Campo de texto para adicionar nova tarefa + botão "Adicionar".
  • Lista (<ul>) exibindo todas as tarefas.
  • Cada tarefa deve ter:
    • Texto da tarefa.
    • Checkbox para marcar como concluída (quando marcado, o texto deve ser riscado com text-decoration: line-through).
    • Botão "Remover" para excluir a tarefa individualmente.
  • Botão "Limpar todas" que remove todas as tarefas.
  • Contador de tarefas pendentes (ex: "3 tarefas restantes").
  • Usar localStorage para salvar as tarefas automaticamente sempre que houver alteração (adicionar, remover, marcar/desmarcar).
  • Carregar as tarefas salvas ao iniciar a página (evento DOMContentLoaded).
🎨 Extras (opcionais, mas recomendados):
  • Permitir editar o texto da tarefa (duplo clique no texto abre um prompt).
  • Filtros: mostrar todas, apenas ativas, apenas concluídas.
  • Animações suaves ao adicionar/remover tarefas (CSS transition).
  • Design responsivo e agradável (uso de CSS Grid/Flexbox).

Estrutura de dados sugerida: Um array de objetos, cada objeto com: { id: Date.now(), texto: "string", concluida: false }.

Este projeto consolida: arrays de objetos, manipulação do DOM, eventos, localStorage e funções de alta ordem (map, filter, forEach).
