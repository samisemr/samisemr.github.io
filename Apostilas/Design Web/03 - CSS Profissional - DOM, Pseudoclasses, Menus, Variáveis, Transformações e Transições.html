1. Árvore do documento (DOM) – a hierarquia dos elementos

O Document Object Model (DOM) é uma representação em árvore de todos os elementos HTML de uma página. Cada elemento é um nó, e as relações entre eles seguem uma estrutura hierárquica: pai, filho, ancestral, descendente, irmão. Entender o DOM é fundamental para aplicar seletores CSS avançados, pois você pode mirar em elementos baseados em sua posição e relação com outros elementos, sem depender de classes ou IDs.

<div class="container">
    <p>Parágrafo filho direto 1</p>
    <div>
        <p>Parágrafo descendente (neto)</p>
    </div>
    <p>Parágrafo filho direto 2</p>
</div>

/* CSS */
.container > p { color: blue; }      /* Apenas os filhos diretos (1 e 2) ficam azuis */
.container p { text-decoration: underline; } /* Todos os parágrafos (incluindo neto) */

h2 + p { margin-top: 0; }            /* Remove margem do parágrafo após um h2 */
h2 ~ p { color: gray; }              /* Todos os parágrafos irmãos após h2 ficam cinza */

1.2 Herança no CSS: o que é passado para os filhos?

Herança é o mecanismo pelo qual algumas propriedades CSS aplicadas a um elemento são automaticamente repassadas para seus descendentes. Isso evita repetição de código. Propriedades que geralmente são herdadas: color, font-family, font-size, line-height, text-align. Propriedades que não são herdadas: width, height, margin, padding, border, background.

💡 Controle fino da herança:
- inherit → força um elemento a herdar o valor do pai.
- initial → define o valor padrão do navegador.
- unset → se a propriedade é herdável, age como inherit; senão, como initial. 
body {
    color: #212529;        /* herdado por todos os textos */
    font-family: 'Arial', sans-serif;
}

/* Forçando herança onde não ocorre naturalmente */
button {
    color: inherit;        /* botão herda a cor do body */
    font-family: inherit;
}

2. Pseudoclasses: estilizando estados do elemento

Pseudoclasses (representadas por :) permitem estilizar um elemento com base em seu estado ou posição na árvore, não apenas em sua tag ou classe. São extremamente úteis para criar interatividade.

Pseudoclasse:hover:focus:active:visited:link:first-child:last-child:nth-child(n):not(seletor):checked:disabled
DescriçãoExemplo de uso real
Mouse sobre o elementoMenu que muda de cor, botão com efeito de elevação
Elemento selecionado (tab ou clique)Campo de formulário com borda azul quando ativo
Elemento sendo clicado (momento do clique)Botão que "afunda" ligeiramente ao clicar
Link já clicado pelo usuárioLinks visitados com cor diferente
Link não visitadoEstilo padrão de links não acessados

Último filho do paiÚltimo cartão de uma galeria sem margem direita
Enésimo filho (1-based)Linhas pares/ímpares de tabela: :nth-child(even)
Exceção – tudo exceto o seletorTodos os botões exceto o último: .btn:not(:last-child)
Checkbox ou radio marcadoPersonalizar o visual de inputs selecionados
Campo desabilitadoEstilizar campos bloqueados com fundo cinza claro
⚠️ Ordem LVHA para links: Para que todos os estados funcionem corretamente, os estilos de links devem seguir a ordem: :link → :visited → :hover → :active. Caso contrário, regras podem ser sobrescritas indevidamente. 
/* Exemplo completo de link */
a:link { color: #0a1c3a; text-decoration: none; }
a:visited { color: #6c757d; }
a:hover { color: #2c7da0; text-decoration: underline; }
a:active { color: #f4a261; }

/* Estilizando linhas alternadas de tabela */
tr:nth-child(even) { background-color: #f8f9fa; }
tr:nth-child(odd) { background-color: #ffffff; }

/* Excluindo o primeiro item */
.item:not(:first-child) { margin-top: 10px; }

2.1 Pseudoelementos: estilizando partes específicas

Pseudoelementos (representados por ::) permitem estilizar partes de um elemento ou inserir conteúdo virtual antes ou depois. Diferente das pseudoclasses, que representam estados, pseudoelementos criam "sub-elementos" lógicos.

Pseudoelemento::before::after::first-letter::first-line::marker::selection
DescriçãoExemplo prático
Insere conteúdo ANTES do conteúdo do elementoAdicionar ícone antes de cada link externo
Insere conteúdo DEPOIS do conteúdo do elementoAdicionar "🔗" após links que abrem em nova aba
Primeira letra do textoEfeito de letra capitular (estilo revista)
Primeira linha do texto (depende do viewport)Destacar a primeira linha de um parágrafo
Marcador de item de lista (bullet ou número)Personalizar cor e tamanho dos bullets
Texto selecionado pelo usuárioAlterar cor de fundo e texto quando o usuário seleciona conteúdo
 
/* Letra capitular estilo livro */
article p:first-of-type::first-letter {
    font-size: 48px;
    font-weight: bold;
    color: #2c7da0;
    float: left;
    margin-right: 10px;
    line-height: 1;
}

/* Adicionar ícone de PDF antes de links para arquivos PDF */
a[href$=".pdf"]::before {
    content: "📄 ";
}

/* Personalizar bullets de lista */
ul li::marker {
    color: #2c7da0;
    font-size: 1.2em;
}

/* Estilo do texto selecionado */
::selection {
    background-color: #2c7da0;
    color: white;
}

3. Barra de navegação profissional

Menus são componentes fundamentais em qualquer site. A melhor prática é construí-los com listas não ordenadas (<ul>), pois semanticamente representam uma lista de links de navegação.

3.1 Menu vertical estilizado

<!-- HTML -->
<nav class="menu-vertical">
    <ul>
        <li><a href="#">Início</a></li>
        <li><a href="#">Sobre</a></li>
        <li><a href="#">Serviços</a></li>
        <li><a href="#">Contato</a></li>
    </ul>
</nav>

/* CSS */
.menu-vertical ul {
    list-style: none;
    margin: 0;
    padding: 0;
    width: 250px;
    background-color: #2c7da0;
    border-radius: 12px;
    overflow: hidden;
}

.menu-vertical li a {
    display: block;        /* Torna toda a área do link clicável */
    color: white;
    padding: 14px 20px;
    text-decoration: none;
    transition: background 0.3s;
}

.menu-vertical li a:hover {
    background-color: #0a1c3a;
}

3.2 Menu horizontal com inline-block e flexbox

A abordagem moderna usa Flexbox, que é mais flexível e poderosa que float ou inline-block.

.menu-horizontal ul {
    list-style: none;
    margin: 0;
    padding: 0;
    background-color: #2c7da0;
    display: flex;          /* Ativa o Flexbox */
    gap: 20px;             /* Espaçamento entre itens */
    padding: 0 20px;
}

.menu-horizontal li a {
    display: block;
    color: white;
    padding: 14px 16px;
    text-decoration: none;
    transition: background 0.3s;
}

.menu-horizontal li a:hover {
    background-color: #0a1c3a;
}

3.3 Menu suspenso (dropdown) com subnível

Dropdowns permitem economizar espaço organizando links relacionados em submenus. A técnica envolve aninhar uma nova lista dentro de um item de menu e usar position: absolute para posicionar o submenu.

<!-- HTML -->
<nav class="dropdown-menu">
    <ul>
        <li><a href="#">Home</a></li>
        <li class="dropdown">
            <a href="#">Serviços ▼</a>
            <ul class="submenu">
                <li><a href="#">Web Design</a></li>
                <li><a href="#">Edição de Vídeo</a></li>
                <li><a href="#">Motion Graphics</a></li>
            </ul>
        </li>
        <li><a href="#">Portfólio</a></li>
        <li><a href="#">Contato</a></li>
    </ul>
</nav>

/* CSS */
.dropdown-menu ul {
    list-style: none;
    margin: 0;
    padding: 0;
    display: flex;
    background-color: #2c7da0;
}

.dropdown-menu li {
    position: relative;    /* Necessário para posicionar o submenu */
}

.dropdown-menu li a {
    display: block;
    color: white;
    padding: 14px 20px;
    text-decoration: none;
}

.submenu {
    position: absolute;
    top: 100%;             /* Aparece abaixo do item pai */
    left: 0;
    background-color: #1e4a6b;
    flex-direction: column;
    min-width: 180px;
    display: none;         /* Escondido por padrão */
    border-radius: 8px;
    overflow: hidden;
}

.dropdown:hover .submenu {
    display: flex;         /* Exibe ao passar o mouse */
}

.submenu li a:hover {
    background-color: #0a1c3a;
}

4. Box-sizing: controlando o dimensionamento

Por padrão, a largura total de um elemento é calculada como: width + padding-left + padding-right + border-left + border-right. Isso frequentemente causa surpresas em layouts, pois um elemento com width: 300px e padding: 20px ocupará 340px na prática.

✅ Solução moderna: box-sizing: border-box; faz com que padding e borda sejam deduzidos da largura definida. Assim, width: 300px sempre será 300px, independente de padding e borda. É altamente recomendável aplicar a todos os elementos: 
* {
    box-sizing: border-box;
}

4.1 Variáveis CSS (Custom Properties) – o poder da manutenção

Variáveis CSS (também chamadas de Custom Properties) permitem armazenar valores reutilizáveis em um único lugar. São declaradas com --nome e usadas com var(--nome). O escopo global é definido no seletor :root (que representa o elemento <html>).

:root {
    --cor-primaria: #2c7da0;
    --cor-secundaria: #0a1c3a;
    --cor-destaque: #f4a261;
    --espacamento-padrao: 16px;
    --border-radius-padrao: 12px;
    --sombra-suave: 0 4px 12px rgba(0,0,0,0.1);
}

.card {
    background: white;
    border-radius: var(--border-radius-padrao);
    padding: var(--espacamento-padrao);
    box-shadow: var(--sombra-suave);
}

.botao {
    background-color: var(--cor-primaria);
    color: white;
    border-radius: calc(var(--border-radius-padrao) / 2);
}

Vantagens das variáveis CSS: mudar um valor no :root atualiza automaticamente todas as ocorrências; facilita criação de temas (ex: tema escuro/claro); pode ser manipulado via JavaScript.

4.2 Formatação profissional de imagens

/* Imagem responsiva (essencial para mobile) */
img {
    max-width: 100%;
    height: auto;
    display: block;
}

/* Imagem circular (avatar) */
.avatar {
    width: 150px;
    height: 150px;
    border-radius: 50%;
    object-fit: cover;   /* Garante que a imagem preencha sem distorcer */
}

/* Galeria de imagens com efeito hover */
.galeria img {
    transition: transform 0.3s, opacity 0.3s;
}

.galeria img:hover {
    transform: scale(1.05);
    opacity: 0.9;
    box-shadow: var(--sombra-suave);
}

5. Transformações 2D: mova, gire, escale

A propriedade transform permite modificar o espaço visual de um elemento sem afetar o fluxo do documento. É excelente para animações e microinterações.

Funçãotranslate(x, y)Move o elemento (x horizontal, y vertical)
DescriçãoExemploEfeito visual
transform: translate(50px, 20px);Desloca 50px para direita, 20px para baixo rotate(deg)Gira o elemento em graus (positivo = horário)transform: rotate(15deg);Inclina 15 graus no sentido horário scale(x, y)Altera o tamanho (1 = original, 1.5 = 50% maior)transform: scale(1.2);Aumenta 20% mantendo proporções skewX(deg)Inclina no eixo X (distorce horizontalmente)transform: skewX(10deg);Efeito de "inclinação" lateral skewY(deg)Inclina no eixo Ytransform: skewY(5deg);Efeito de inclinação vertical 
/* Múltiplas transformações (a ordem importa!) */
.card:hover {
    transform: translateY(-8px) scale(1.02) rotate(1deg);
}

/* Ícone que gira ao passar o mouse */
.icone-giratorio:hover {
    transform: rotate(360deg);
    transition: transform 0.5s ease;
}
⚠️ A ordem das transformações importa! transform: translate(50px) rotate(45deg) é diferente de rotate(45deg) translate(50px). No primeiro, move primeiro e depois gira; no segundo, gira primeiro e depois move (o eixo de movimento também gira).

5.1 Transições suaves: o segredo da interatividade

transition cria animações automáticas quando uma propriedade muda de valor (ex: no hover). Sem transição, as mudanças são instantâneas; com transição, tornam-se suaves e profissionais.

.elemento {
    background-color: #2c7da0;
    width: 200px;
    height: 200px;
    transition: all 0.3s ease-in-out;
    /* Propriedades individuais: transition-property, transition-duration, transition-timing-function, transition-delay */
}

.elemento:hover {
    background-color: #0a1c3a;
    width: 250px;
}

/* Timing functions úteis */
.transicao-linear { transition-timing-function: linear; }      /* Velocidade constante */
.transicao-ease { transition-timing-function: ease; }         /* Acelera e desacelera suavemente */
.transicao-ease-in { transition-timing-function: ease-in; }   /* Começa devagar, acelera no fim */
.transicao-ease-out { transition-timing-function: ease-out; } /* Começa rápido, desacelera */
.transicao-custom { transition-timing-function: cubic-bezier(0.68, -0.55, 0.265, 1.55); } /* Efeito elástico */

5.2 Combinando transformações e transições – efeitos avançados

/* Card com efeito 3D sutil */
.card-interativo {
    transition: all 0.3s cubic-bezier(0.2, 0.9, 0.4, 1.1);
}

.card-interativo:hover {
    transform: translateY(-10px) scale(1.02);
    box-shadow: 0 20px 30px -10px rgba(0,0,0,0.2);
}

/* Botão com efeito pulsante (combinação de transformações) */
.botao-pulsante {
    animation: pulse 2s infinite;
}

@keyframes pulse {
    0% { transform: scale(1); }
    50% { transform: scale(1.05); }
    100% { transform: scale(1); }
}

6. Resumo prático para consulta rápida

  • ✅ DOM: Hierarquia pai-filho-irmão. Seletores: > (filho direto), + (irmão adjacente), ~ (irmãos posteriores).
  • ✅ Pseudoclasses principais: :hover, :focus, :first-child, :nth-child(), :not().
  • ✅ Pseudoelementos úteis: ::before, ::after (inserir conteúdo), ::first-letter, ::selection.
  • ✅ Menus: Construir com <ul>; dropdown com position: absolute + display: none/hover; Flexbox para layout horizontal.
  • ✅ Box-sizing: border-box evita surpresas com padding e borda.
  • ✅ Variáveis CSS: :root { --nome: valor; } e var(--nome).
  • ✅ Transformações 2D: translate(), rotate(), scale(), skew().
  • ✅ Transições: transition: propriedade duração timing; – use all 0.3s ease para efeitos gerais.
