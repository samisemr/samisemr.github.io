1. JavaScript: o que é, onde roda e para que serve

JavaScript (JS) é uma linguagem de programação de alto nível, interpretada e multiparadigma. Originalmente criada para rodar em navegadores (lado do cliente), hoje também é usada no servidor (Node.js), em aplicações mobile (React Native), desktop (Electron) e até em IoT. Diferente de HTML e CSS (que são linguagens de marcação e estilo), o JavaScript é uma linguagem de programação completa, com variáveis, funções, estruturas de controle, arrays, objetos e muito mais.

📌 O que torna JavaScript essencial para o desenvolvimento web moderno?
- Manipulação do DOM: alterar conteúdo, estilos e estrutura da página dinamicamente.
- Eventos: reagir a cliques, movimentos do mouse, teclas pressionadas, carregamento da página.
- Comunicação assíncrona (AJAX/Fetch): buscar dados do servidor sem recarregar a página.
- Validação de formulários antes do envio.
- Animações, jogos, aplicações complexas (Single Page Applications - SPAs).

1.1 Como o navegador executa JavaScript

O navegador possui um motor JavaScript (chamado V8 no Chrome, SpiderMonkey no Firefox, JavaScriptCore no Safari) que:

  1. Lê o código-fonte (arquivo .js ou bloco <script>).
  2. Converte em uma representação intermediária (bytecode).
  3. Executa as instruções linha por linha (interpretação) – embora motores modernos usem compilação JIT (Just-In-Time) para performance.

1.2 Onde colocar o código (interno vs externo)

Método
SintaxeVantagensDesvantagens
JavaScript Interno <script> // código aqui </script> (geralmente no final do <body>) Bom para protótipos e exemplos rápidos, não requer arquivos adicionais Não reutilizável entre páginas, mistura estrutura com comportamento
JavaScript Externo (recomendado) <script src="script.js"></script> (arquivo separado) Reutilizável em várias páginas, cacheável pelo navegador, código mais organizado Requer requisição HTTP extra (mínima)
Inline (eventos HTML) <button onclick="alert('Oi')">Clique</button> Rápido para testes Mistura HTML com JS (péssima prática), difícil manutenção
⚠️ Posicionamento da tag <script>:
Colocar scripts no <head> pode bloquear o carregamento da página (o navegador para de renderizar até baixar/executar o JS). A prática recomendada é colocar os scripts no final do <body> ou usar os atributos defer ou async. 
<!-- Posicionamento recomendado -->
<body>
    <h1>Minha Página</h1>
    <!-- todo o conteúdo HTML -->
    
    <script src="meu-script.js"></script>
</body>

<!-- Com defer (executa após o HTML ser parseado) -->
<script src="script.js" defer></script>

<!-- Com async (baixa em paralelo e executa assim que pronto) -->
<script src="analytics.js" async></script>

2. Variáveis, tipos de dados e constantes

Variáveis são "caixas" onde armazenamos valores que podem mudar durante a execução do programa. Constantes guardam valores fixos.

2.1 Declarando variáveis: let, const e o obsoleto var

Palavra-chave
EscopoPode ser reatribuída?Pode ser redeclarada?Recomendação
let Bloco { ... } Sim Não (no mesmo escopo) Use para variáveis que vão mudar de valor
const Bloco { ... } Não (gera erro) Não Use para valores fixos (PI, URL da API, etc.)
var (obsoleto) Função (ou global) Sim Sim EVITE – causa bugs de escopo e hoisting confuso 
"use strict"; // Modo estrito – ajuda a evitar erros comuns

// Declarações corretas
let nome = "João";
let idade = 25;
const PI = 3.14159;
const URL_API = "https://api.exemplo.com";

// Reatribuição permitida (com let)
nome = "Maria";   // OK
idade = idade + 1;

// Reatribuição NÃO permitida (com const)
// PI = 3.14;     // ERRO! Assignment to constant variable

// Nomes de variáveis: case-sensitive, podem conter letras, números, $, _
let precoProduto = 99.90;   // camelCase (padrão JavaScript)
let _privado = "dado";      // underline para "privado" (convenção)
let $elemento = document.querySelector("#id");

2.3 Operadores essenciais

Categoria
OperadoresExemploResultado

🔍 Diferença crucial: == vs ===
- == (igualdade abstrata): tenta converter os tipos antes de comparar. Ex: 5 == "5" → true (string "5" vira número 5).
- === (igualdade estrita): compara valor E tipo sem conversão. Ex: 5 === "5" → false (number vs string).
✅ Sempre prefira === e !== – evita bugs silenciosos.

3. Interação com o usuário: alert, confirm, prompt

JavaScript oferece três funções nativas para comunicação básica com o usuário. São bloqueantes (param a execução do script até o usuário responder) e úteis para aprendizado, mas em projetos reais prefira interfaces HTML personalizadas.

Função
O que fazRetornoExemplo
alert(mensagem) Exibe uma caixa de diálogo com uma mensagem e um botão "OK" undefined (não retorna valor) alert("Bem-vindo ao site!");
confirm(pergunta) Exibe uma caixa com mensagem, botões "OK" (true) e "Cancelar" (false) boolean (true se OK, false se Cancelar) let maiorIdade = confirm("Você tem 18 anos ou mais?");
prompt(pergunta, valorPadrao) Exibe uma caixa com campo de texto para o usuário digitar string (o que o usuário digitou) ou null (se cancelar) let nome = prompt("Qual é o seu nome?", "Visitante"); 
// Exemplo completo usando as três funções
let nome = prompt("Digite seu nome:", "Anônimo");

if (nome !== null && nome.trim() !== "") {
    let confirmacao = confirm(`Olá, ${nome}! Deseja receber nossa newsletter?`);
    
    if (confirmacao) {
        alert(`Ótimo! ${nome}, você será inscrito.`);
    } else {
        alert(`Tudo bem, ${nome}. Você pode se inscrever depois.`);
    }
} else {
    alert("Você não digitou um nome válido.");
}

4. Estruturas condicionais: if/else, else if, switch

Estruturas condicionais permitem que o programa execute diferentes blocos de código baseado em condições lógicas (verdadeiro/falso).

4.1 if, else if, else

let idade = Number(prompt("Digite sua idade:"));

if (isNaN(idade)) {
    console.log("Valor inválido. Digite um número.");
} else if (idade < 12) {
    console.log("Criança");
} else if (idade < 18) {
    console.log("Adolescente");
} else if (idade < 60) {
    console.log("Adulto");
} else {
    console.log("Idoso");
}

// Operador ternário (forma reduzida do if/else)
let status = idade >= 18 ? "Maior de idade" : "Menor de idade";
console.log(status);

4.2 switch – múltiplas comparações com um mesmo valor

let diaSemana = prompt("Digite um dia da semana (segunda, terça, ...):").toLowerCase();

switch (diaSemana) {
    case "segunda":
    case "terça":
    case "quarta":
    case "quinta":
    case "sexta":
        console.log("Dia útil - hora de trabalhar!");
        break;
    case "sábado":
    case "domingo":
        console.log("Fim de semana - descanso!");
        break;
    default:
        console.log("Dia inválido. Digite um dia da semana válido.");
}

5. Laços de repetição (loops)

Loops executam um bloco de código repetidamente enquanto uma condição for verdadeira.

Loop
SintaxeQuando usarExemplo
for for (inicialização; condição; incremento) { } Quando se sabe (ou pode calcular) o número de repetições for (let i = 0; i < 10; i++) { console.log(i); }
while while (condição) { } Quando o número de repetições é desconhecido, e depende de uma condição que pode mudar dentro do loop while (senha !== "1234") { senha = prompt("Digite a senha:"); }
do...while do { } while (condição); Quando o bloco deve ser executado PELO MENOS uma vez (a condição é verificada no final) do { resposta = prompt("Continuar? (s/n)"); } while (resposta === "s"); 
// Exemplo 1: Tabuada com for
let numero = 7;
console.log(`Tabuada do ${numero}:`);
for (let i = 1; i <= 10; i++) {
    console.log(`${numero} x ${i} = ${numero * i}`);
}

// Exemplo 2: while com contador
let contador = 0;
while (contador < 5) {
    console.log(`Volta número ${contador + 1}`);
    contador++;
}

// Exemplo 3: do...while (executa ao menos uma vez)
let opcao;
do {
    opcao = prompt("Escolha: 1 - Ver saldo, 2 - Sair");
    if (opcao === "1") {
        alert("Seu saldo é R$ 1.000,00");
    }
} while (opcao !== "2");
alert("Sistema encerrado.");

// Exemplo 4: break (interrompe o loop) e continue (pula para próxima iteração)
for (let i = 1; i <= 10; i++) {
    if (i === 5) {
        console.log("Parou no 5!");
        break;  // sai do loop
    }
    console.log(i);
}

for (let i = 1; i <= 10; i++) {
    if (i % 2 === 0) {
        continue;  // pula números pares
    }
    console.log(i);  // exibe apenas ímpares
}

6. Manipulação básica do DOM (Document Object Model)

O DOM é uma representação em árvore de todos os elementos HTML da página. Com JavaScript, podemos acessar, modificar, adicionar ou remover elementos dinamicamente.

Representação da árvore DOM: document → html → head/body → elementos filhos

6.3 Funções: reutilizando código

Funções são blocos de código nomeados que podem ser executados quantas vezes quisermos. Elas evitam repetição e organizam a lógica.

// Declaração de função (function declaration)
function saudacao(nome) {
    return `Olá, ${nome}! Seja bem-vindo.`;
}

// Chamando a função
let mensagem = saudacao("Carlos");
console.log(mensagem);  // "Olá, Carlos! Seja bem-vindo."

// Função com parâmetros padrão (ES6)
function calcularPrecoComDesconto(preco, desconto = 0.1) {
    return preco - (preco * desconto);
}
console.log(calcularPrecoComDesconto(100));      // 90 (desconto padrão 10%)
console.log(calcularPrecoComDesconto(100, 0.2)); // 80 (desconto 20%)

// Arrow function (sintaxe moderna)
const somar = (a, b) => a + b;
console.log(somar(5, 3));  // 8

// Função anônima (evento)
document.getElementById("meuBotao").onclick = function() {
    alert("Botão clicado!");
};

7. Boas práticas e dicas finais

  • Sempre use "use strict" no início dos seus scripts (ou dentro de funções) para evitar variáveis globais acidentais e erros silenciosos.
  • Nomeie variáveis e funções de forma clara em inglês ou português (mas seja consistente). Ex: calcularTotal, usuarioEstaLogado.
  • Use === e !== para comparações. Evite == e !=.
  • Evite poluição do escopo global: mantenha variáveis dentro de funções ou blocos (let e const têm escopo de bloco).
  • Comente seu código explicando o "porquê", não o "o quê" (o código já diz o que faz).
  • Use o console do navegador para testar trechos antes de integrar ao projeto.
  • Mantenha o JavaScript separado do HTML (arquivos .js externos) para facilitar manutenção.
🧠 Desafio final (projeto prático):

- Permita que o usuário faça novos cálculos sem recarregar a página (use um loop ou um botão "Novo cálculo").
