Layouts Modernos: HTML Semântico, Grid, Flexbox e Responsividade
Construa sites adaptáveis a qualquer dispositivo com as ferramentas atuais do mercado

Chegamos ao nível profissional: você vai aprender a estruturar páginas com elementos semânticos do HTML5 (header, nav, section, article, aside, footer), dominar dois poderosos sistemas de layout – CSS Grid (bidimensional) e Flexbox (unidimensional) – e tornar tudo responsivo com Media Queries e imagens flexíveis. 1. HTML Semântico: estruturando com significado

No passado, usávamos <div> para tudo. Hoje, o HTML5 oferece elementos que descrevem o que cada parte do conteúdo representa. Isso melhora a acessibilidade, o SEO e a manutenção do código.

1.1 Principais elementos estruturais

ElementoSignificadoUso típico
<header>Cabeçalho da página ou seçãoLogotipo, título, navegação principal
<nav>Navegação principalMenu de links para outras páginas
<main>Conteúdo principal únicoParte central e exclusiva da página
<section>Agrupamento temático de conteúdoSeção de serviços, diferenciais, notícias
<article>Conteúdo independente e reutilizávelPost de blog, comentário, card de produto
<aside>Conteúdo relacionado (barra lateral)Links extras, propaganda, biografia
<footer>Rodapé da página ou seçãoCopyright, contato, links legais
<hgroup>Agrupa título e subtítulo<h1>Título</h1><h2>Subtítulo</h2>
📌 Regra de ouro: Use <div> apenas quando nenhum outro elemento semântico for adequado. Prefira sempre <section>, <article>, <nav> etc. para dar significado ao conteúdo.

1.2 Exemplo de estrutura semântica completa

<body>
  <header>
    <h1>Meu Site</h1>
    <nav>
      <ul><li><a href="#">Home</a></li><li><a href="#">Sobre</a></li></ul>
    </nav>
  </header>
  
  <main>
    <section>
      <h2>Nossos Serviços</h2>
      <article>Serviço 1</article>
      <article>Serviço 2</article>
    </section>
    <aside>
      <h3>Links Úteis</h3>
    </aside>
  </main>
  
  <footer>
    <p>&copy; 2025 - Todos os direitos reservados</p>
  </footer>
</body>

Pegue uma página antiga cheia de <div> com classes como "cabecalho", "menu", "conteudo-principal", "sidebar", "rodape". Substitua pelos elementos semânticos correspondentes (header, nav, main, section, aside, footer). Visualize e veja que o layout permanece igual, mas o código ficou muito mais significativo.

2. CSS Grid Layout – layout bidimensional poderoso

Grid é ideal para criar layouts complexos em duas dimensões (linhas E colunas simultaneamente). Perfeito para estruturar a página inteira.

2.1 Conceitos fundamentais

  • Grid container: elemento com display: grid;
  • Grid item: filhos diretos do container.
  • Linhas (rows) e colunas (columns): definidas com grid-template-rows e grid-template-columns.
  • Gap: espaçamento entre células (gap: 20px;).
  • Unidade fr: fração do espaço disponível. Ex: 1fr 2fr.

2.2 Exemplo prático: layout clássico de página

.container {
    display: grid;
    grid-template-columns: 250px 1fr 250px;  /* 3 colunas: laterais fixas, centro flexível */
    grid-template-rows: auto 1fr auto;       /* header, conteúdo principal, footer */
    gap: 20px;
    min-height: 100vh;
}
header { grid-column: 1 / -1; }  /* ocupa todas as colunas */
nav { grid-column: 1 / 2; }
main { grid-column: 2 / 3; }
aside { grid-column: 3 / 4; }
footer { grid-column: 1 / -1; }

2.3 Nomeando áreas com grid-template-areas

É a forma mais intuitiva de criar layouts com Grid.

.container {
    display: grid;
    grid-template-areas: 
        "header header header"
        "nav main aside"
        "footer footer footer";
    grid-template-columns: 200px 1fr 200px;
    gap: 15px;
}
header { grid-area: header; }
nav { grid-area: nav; }
main { grid-area: main; }
aside { grid-area: aside; }
footer { grid-area: footer; }

2.4 Funções úteis: repeat(), minmax() e calc()

  • repeat(3, 1fr) → cria 3 colunas com 1fr cada.
  • minmax(200px, auto) → coluna com largura mínima de 200px e máxima automática.
  • calc(100% - 40px) → calcula valores dinamicamente.

📐 Micro-atividade 2: Página inteira com Grid

Recrie o layout de um portal de notícias usando Grid: header com logo/menu, nav lateral com categorias, main com artigos em grade, aside com anúncios, footer com informações. Use grid-template-areas e torne o layout responsivo (ajustando as áreas para mobile depois).

3. Flexbox – layout unidimensional para componentes

Enquanto o Grid é ótimo para o layout geral da página, o Flexbox é perfeito para alinhar e distribuir espaços dentro de um componente, como cards, menus, barras de ferramentas e listas horizontais.

3.1 Ativando o Flexbox

.flex-container {
    display: flex;
    flex-direction: row;  /* row (padrão), column, row-reverse, column-reverse */
    justify-content: space-between;  /* alinhamento horizontal */
    align-items: center;  /* alinhamento vertical */
    flex-wrap: wrap;  /* permite quebra de linha */
}

3.2 Propriedades essenciais

PropriedadeO que fazValores comuns
justify-contentAlinha itens no eixo principal (horizontal em row)flex-start, center, space-between, space-around
align-itemsAlinha itens no eixo transversal (vertical em row)flex-start, center, stretch, baseline
flex-directionDefine direção principalrow, column, row-reverse
flex-wrapPermite ou não quebra de linhanowrap, wrap
gapEspaçamento entre itens10px, 1rem

3.3 Exemplo: menu horizontal e cards flexíveis

/* Menu horizontal */
.menu {
    display: flex;
    justify-content: center;
    gap: 30px;
    background: #243E73;
    padding: 15px;
}
/* Cards que se ajustam */
.cards {
    display: flex;
    flex-wrap: wrap;
    gap: 20px;
    justify-content: center;
}
.card {
    flex: 1 1 250px;  /* cresce, encolhe, base 250px */
    background: #F2D8CE;
    padding: 20px;
    border-radius: 12px;
}

🧩 Micro-atividade 3: Galeria flexível

Crie uma galeria de produtos usando Flexbox. Os produtos devem aparecer em linhas, com quantidade variável por linha (usando flex-wrap). Cada card deve ter largura mínima de 220px e máxima de 1fr. Centralize a galeria e adicione espaçamento entre os cards com gap.

4. Media Queries – responsividade para todos os dispositivos

Media Queries permitem aplicar CSS diferente dependendo das características do dispositivo, principalmente a largura da tela. É o coração do design responsivo.

4.1 Sintaxe básica

/* Para telas com largura máxima de 768px (tablets e celulares) */
@media (max-width: 768px) {
    .container {
        grid-template-columns: 1fr;
    }
    .menu {
        flex-direction: column;
    }
}

/* Para telas entre 769px e 1024px */
@media (min-width: 769px) and (max-width: 1024px) {
    .container {
        grid-template-columns: 1fr 1fr;
    }
}

4.2 Breakpoints mais comuns no mercado

  • Mobile: até 600px
  • Tablet: 601px a 900px
  • Desktop pequeno: 901px a 1200px
  • Desktop grande: acima de 1200px

4.3 Estratégia Mobile First

Comece o CSS com estilos para dispositivos móveis (padrão) e use min-width para adicionar melhorias em telas maiores. Isso é mais performático e recomendado pelo Google.

/* Estilos base (mobile) */
.container { display: flex; flex-direction: column; }
/* Tablet e acima */
@media (min-width: 768px) {
    .container { flex-direction: row; flex-wrap: wrap; }
}
/* Desktop */
@media (min-width: 1024px) {
    .container { max-width: 1200px; margin: auto; }
}

📱 Micro-atividade 4: Adaptando layout com Media Queries

Pegue o layout com Grid que você criou na atividade 2 e torne-o responsivo: em telas menores que 768px, transforme as 3 colunas em 1 coluna (grid-template-areas deve virar apenas "header", "nav", "main", "aside", "footer" empilhados). Teste redimensionando o navegador.

5. Imagens responsivas e elementos picture

Imagens são parte crucial do design responsivo. Use as técnicas abaixo para garantir que elas carreguem bem e se adaptem.

5.1 Imagem fluida com CSS

img {
    max-width: 100%;
    height: auto;
    display: block;
    margin: 0 auto;  /* centraliza */
}

Isso faz com que a imagem nunca ultrapasse o container e mantenha a proporção.

5.2 Atributo srcset – imagens diferentes por resolução

Permite ao navegador escolher a imagem mais adequada conforme a largura da tela ou densidade de pixels.

<img src="imagem-pequena.jpg"
     srcset="imagem-media.jpg 800w,
             imagem-grande.jpg 1200w"
     sizes="(max-width: 600px) 100vw, 50vw"
     alt="Descrição">

5.3 Elemento <picture> – direção de arte

Quando a imagem precisa ser cortada ou reposicionada em diferentes telas (art direction), use <picture> com múltiplos <source>.

<picture>
  <source media="(min-width: 1024px)" srcset="imagem-desktop.jpg">
  <source media="(min-width: 768px)" srcset="imagem-tablet.jpg">
  <img src="imagem-mobile.jpg" alt="Exemplo de direção de arte">
</picture>

5.4 Background responsivo com CSS

.hero {
    background-image: url('mobile.jpg');
    background-size: cover;
    background-position: center;
}
@media (min-width: 768px) {
    .hero { background-image: url('desktop.jpg'); }
}

🖼️ Micro-atividade 5: Galeria com picture e srcset

Crie uma página que exiba uma imagem de banner. Use <picture> para fornecer três versões: uma para celular (cortada no centro), uma para tablet (mais larga) e uma para desktop (cena completa). Teste redimensionando a janela e veja a imagem mudando nos breakpoints.

6. Juntando tudo: projeto responsivo completo

Um site profissional combina HTML semântico, Grid ou Flexbox (ou ambos) e Media Queries. Exemplo de estrutura híbrida:

<body>
  <div class="grid-container">
    <header>...</header>
    <nav>...</nav>
    <main>
      <div class="cards-flex">  <!-- Flexbox para os cards -->
        <article>Card 1</article>
        <article>Card 2</article>
      </div>
    </main>
    <aside>...</aside>
    <footer>...</footer>
  </div>
</body>
🧠 Dica profissional: Use Grid para o macro layout (estrutura da página) e Flexbox para componentes internos (cards, listas, barras). Essa combinação é a mais usada no mercado atualmente.

🎯 PROJETO FINAL – Site responsivo completo

Desafio: Construa um site de portfólio ou landing page para uma agência criativa. Requisitos obrigatórios:

  • HTML 100% semântico (header, nav, main, section, article, aside, footer).
  • Layout principal com CSS Grid (3 colunas em desktop, 1 coluna em mobile).
  • Cards de serviços/projetos usando Flexbox (flex-wrap, gap, centralização).
  • Menu de navegação responsivo (horizontal em desktop, vertical/empilhado em mobile via media query).
  • Imagem de banner com <picture> ou srcset adaptada para 3 tamanhos.
  • Media queries com pelo menos dois breakpoints (tablet e desktop).
