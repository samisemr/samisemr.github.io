1. HTML Semântico: estruturando com significado

Antes do HTML5, desenvolvedores usavam <div> para absolutamente tudo, com classes como class="header", class="sidebar". Isso funcionava, mas não transmitia significado para navegadores, leitores de tela ou mecanismos de busca. O HTML5 introduziu elementos semânticos – tags que descrevem o que o conteúdo representa, não apenas como ele aparece.

📌 Por que usar HTML semântico?
- Acessibilidade: Leitores de tela (usados por pessoas com deficiência visual) navegam melhor pela página.
- SEO (Search Engine Optimization): Google e outros mecanismos de busca entendem melhor a hierarquia do conteúdo.
- Manutenção: Código mais limpo e fácil de entender para outros desenvolvedores.
- Performance: Navegadores podem otimizar o carregamento de elementos semânticos.

1.1 Principais elementos estruturais

Elemento

<nav> Navegação principal do site Menu com links para outras páginas ou seções <div class="nav">
<main> Conteúdo principal e único da página Parte central que não se repete entre páginas <div class="main">
<section> Agrupamento temático de conteúdo Seção de serviços, diferenciais, notícias, contato <div class="section">
<article> Conteúdo independente e reutilizável Post de blog, comentário, card de produto, notícia <div class="article">
<aside> Conteúdo relacionado (barra lateral) Links extras, propaganda, biografia do autor, widgets <div class="sidebar">
<footer> Rodapé da página ou seção Copyright, informações de contato, links legais <div class="footer">

🏗️ Micro-atividade 1: Transformando divs em semântica

Pegue uma página HTML antiga (pode ser a que você criou nas apostilas anteriores) que use apenas <div> com classes como "cabecalho", "menu", "conteudo-principal", "sidebar", "rodape". Substitua cada <div> pelo elemento semântico correspondente. Visualize no navegador – o layout continua igual, mas o código ficou muito mais significativo e acessível.

2. CSS Grid Layout – layout bidimensional poderoso

O CSS Grid Layout é um sistema de layout bidimensional, ou seja, controla linhas e colunas simultaneamente. Diferente de outras técnicas que exigiam hacks (floats, tables, inline-block), o Grid foi criado especificamente para construir layouts complexos de forma intuitiva.

2.1 Conceitos fundamentais

  • Grid Container: elemento pai com display: grid; – ativa o contexto grid.
  • Grid Items: os filhos diretos do container, que se tornam células organizadas em linhas e colunas.
  • Grid Tracks: as linhas (rows) e colunas (columns) definidas.
  • Grid Cell: a interseção entre uma linha e uma coluna.
  • Unidade fr (fraction): representa uma fração do espaço disponível. Ex: 1fr 2fr – a segunda coluna terá o dobro da primeira.
  • Gap (ou gutter): espaçamento entre as células, definido por gap: 20px;.
Diagrama explicativo do CSS Grid: grid container, grid items, tracks, cells e gaps
/* Exemplo básico: grid de 3 colunas e 2 linhas */
.container {
    display: grid;
    grid-template-columns: 200px 1fr 200px;  /* coluna fixa | flexível | coluna fixa */
    grid-template-rows: auto 1fr auto;       /* header | conteúdo principal | footer */
    gap: 20px;
    min-height: 100vh;                       /* ocupa 100% da altura da viewport */
}

/* Posicionamento manual dos itens */
header { grid-column: 1 / -1; }  /* ocupa da linha 1 até a última (-1) */
nav    { grid-column: 1 / 2; }
main   { grid-column: 2 / 3; }
aside  { grid-column: 3 / 4; }
footer { grid-column: 1 / -1; }
📐 Unidade fr vs porcentagem vs px:
- px: valor fixo, não se adapta.
- %: relativo ao container, mas soma de porcentagens pode ultrapassar 100%.
- fr: distribui o espaço restante proporcionalmente. Ex: 1fr 2fr 1fr – a coluna do meio terá o dobro das laterais. É a unidade mais flexível e recomendada para layouts responsivos.

2.2 Nomeando áreas com grid-template-areas

Esta é a forma mais intuitiva e legível de criar layouts com Grid. Você desenha o layout literalmente no CSS usando strings.

.layout {
    display: grid;
    grid-template-areas: 
        "header header header"
        "nav main aside"
        "footer footer footer";
    grid-template-columns: 250px 1fr 250px;
    grid-template-rows: auto 1fr auto;
    gap: 16px;
    min-height: 100vh;
}

header { grid-area: header; }
nav    { grid-area: nav; }
main   { grid-area: main; }
aside  { grid-area: aside; }
footer { grid-area: footer; }

/* No responsivo, basta redefinir as áreas */
@media (max-width: 768px) {
    .layout {
        grid-template-areas: 
            "header"
            "nav"
            "main"
            "aside"
            "footer";
        grid-template-columns: 1fr;
    }
}
💡 Funções úteis do Grid:
- repeat(3, 1fr) → cria 3 colunas com 1fr cada. Equivalente a 1fr 1fr 1fr.
- minmax(200px, auto) → define uma largura mínima de 200px e máxima automática.
- auto-fill e auto-fit → usados com repeat para criar grids responsivas sem media queries.
- Exemplo: grid-template-columns: repeat(auto-fit, minmax(250px, 1fr)); – colunas de no mínimo 250px que se ajustam automaticamente.

📐 Micro-atividade 2: Página inteira com Grid

Recrie o layout de um portal de notícias ou blog usando Grid:

  • Header com título e navegação.
  • Nav lateral (ou superior, como preferir) com categorias.
  • Main com uma grade de artigos (use um grid interno com repeat(auto-fit, minmax(280px, 1fr))).
  • Aside com anúncios ou links úteis.
  • Footer com informações de copyright.
  • Utilize grid-template-areas para definir o layout.
  • Adicione uma media query para tablets (max-width: 768px) empilhando todas as áreas.

3. Flexbox – layout unidimensional para componentes

Enquanto o Grid é ideal para o macro layout (estrutura geral da página), o Flexbox é a ferramenta perfeita para componentes internos – alinhar itens em uma única direção (linha ou coluna), distribuir espaço e criar layouts flexíveis.

🔄 Flexbox vs Grid: quando usar cada um?
- Flexbox (1D): ideal para menus, barras de ferramentas, listas horizontais/verticais, cards que precisam quebrar em linhas (com flex-wrap), alinhamento de itens dentro de um container.
- Grid (2D): ideal para o layout principal da página, galerias com alinhamento preciso em linhas e colunas, formulários complexos com várias colunas.
- Combinação vencedora: use Grid para a estrutura externa e Flexbox para os componentes internos (cards, menus, formulários).

3.1 Propriedades essenciais do Flexbox

Propriedade (no container)
O que fazValores comuns
flex-direction Define a direção principal (eixo principal) row (padrão, linha horizontal), column (coluna vertical), row-reverse, column-reverse
justify-content Alinha os itens no eixo principal flex-start, center, flex-end, space-between, space-around, space-evenly
align-items Alinha os itens no eixo transversal flex-start, center, flex-end, stretch (padrão), baseline
flex-wrap Permite ou não que os itens quebrem para a próxima linha nowrap (padrão), wrap, wrap-reverse
gap Espaçamento entre os itens (funciona também no Grid) 10px, 1rem, 20px
Propriedade (nos itens)
O que fazValores comuns
flex Atalho para flex-grow, flex-shrink e flex-basis 1 (cresce igualmente), 0 0 auto (não cresce nem encolhe), 1 1 250px (cresce, encolhe, base 250px)
align-self Sobrescreve o align-items do container para um item específico flex-start, center, flex-end, stretch
order Altera a ordem visual do item (valor numérico) 1, 2, -1 (vem antes)
Visualização dos eixos no Flexbox: main axis e cross axis
/* Exemplo 1: Menu horizontal centralizado */
.menu {
    display: flex;
    justify-content: center;   /* centraliza os itens horizontalmente */
    gap: 30px;                 /* espaçamento entre os links */
    background-color: #0a1c3a;
    padding: 16px;
}
.menu a {
    color: white;
    text-decoration: none;
    padding: 8px 16px;
    transition: background 0.3s;
}
.menu a:hover {
    background-color: #2c7da0;
    border-radius: 8px;
}

/* Exemplo 2: Galeria de cards flexível (quebra de linha automática) */
.galeria {
    display: flex;
    flex-wrap: wrap;      /* permite quebra de linha */
    gap: 20px;
    justify-content: center;
}
.card {
    flex: 1 1 250px;      /* cresce, encolhe, base mínima 250px */
    background-color: #f8f9fa;
    border-radius: 16px;
    padding: 20px;
    text-align: center;
    transition: transform 0.3s;
}
.card:hover {
    transform: translateY(-5px);
}

/* Exemplo 3: Barra de ferramentas com itens alinhados verticalmente */
.toolbar {
    display: flex;
    align-items: center;    /* centraliza verticalmente */
    justify-content: space-between; /* espaça os itens: logo no início, ações no fim */
    background-color: #e9ecef;
    padding: 12px 24px;
}

🧩 Micro-atividade 3: Galeria flexível de produtos

Crie uma galeria de produtos usando Flexbox. Requisitos:

  • Os produtos devem aparecer em linhas com quantidade variável por linha (use flex-wrap: wrap).
  • Cada card deve ter largura mínima de 220px, máxima de 1fr (use flex: 1 1 220px).
  • Centralize a galeria horizontalmente com justify-content: center.
  • Adicione espaçamento entre os cards com gap: 20px.
  • Cada card deve conter: imagem placeholder, título, descrição resumida e preço.
  • No hover do card, aplique um leve aumento de escala (transform: scale(1.02)) e sombra.

4. Media Queries – responsividade para todos os dispositivos

Media Queries são o mecanismo que permite aplicar regras CSS condicionalmente, baseado nas características do dispositivo (principalmente a largura da tela). Sem media queries, não existe design responsivo de verdade.

4.1 Breakpoints mais comuns e estratégia Mobile First

Categoria
Breakpoint (largura)Alvo
Mobile (pequeno)até 480pxSmartphones antigos ou muito compactos
Mobile (grande)481px - 768pxSmartphones modernos em retrato
Tablet769px - 1024pxiPads, tablets Android em retrato/paisagem
Desktop pequeno1025px - 1200pxNotebooks, monitores menores
Desktop grande1201px - 1440pxMonitores padrão
Ultrawide1441px+Monitores widescreen, TVs
📱 Mobile First – a estratégia recomendada pelo Google
Consiste em escrever os estilos base para dispositivos móveis primeiro (sem media query, ou com max-width muito baixo), e depois usar min-width para adicionar melhorias em telas maiores. Vantagens: performance (menos CSS para baixar em celulares) e foco no conteúdo essencial.

Exemplo de Mobile First: 
/* Estilos base para celular (padrão) */
.container {
    display: flex;
    flex-direction: column;
}
.card {
    width: 100%;
}

/* Tablet e acima – adiciona layout em colunas */
@media (min-width: 768px) {
    .container {
        flex-direction: row;
        flex-wrap: wrap;
    }
    .card {
        width: calc(50% - 20px);
    }
}

/* Desktop – mais colunas */
@media (min-width: 1024px) {
    .card {
        width: calc(33.333% - 20px);
    }
}
/* Sintaxe completa de Media Query */
@media (tipo de mídia) and (condição) {
    /* regras CSS */
}

/* Tipos de mídia comuns */
@media screen { ... }      /* telas de computadores, tablets, smartphones */
@media print { ... }       /* impressão - muito útil para esconder menus e ajustar layout */
@media all { ... }         /* todos os tipos (padrão) */

/* Exemplos práticos */
@media (max-width: 768px) {
    /* telas menores que 768px */
    .menu { flex-direction: column; }
    .sidebar { display: none; }
}

@media (min-width: 1024px) and (max-width: 1366px) {
    /* telas entre 1024px e 1366px */
    .container { max-width: 960px; margin: 0 auto; }
}

@media (orientation: portrait) {
    /* orientação retrato (altura > largura) */
    .banner { background-image: url('banner-portrait.jpg'); }
}

@media (orientation: landscape) {
    /* orientação paisagem (largura > altura) */
    .banner { background-image: url('banner-landscape.jpg'); }
}

📱 Micro-atividade 4: Adaptando layout com Media Queries

Pegue o layout com Grid que você criou na Micro-atividade 2 e torne-o totalmente responsivo:

  • Em telas menores que 768px (mobile): transforme o grid de 3 colunas em 1 coluna, com todas as áreas empilhadas (header, nav, main, aside, footer).
  • Em telas entre 768px e 1024px (tablet): mantenha o grid de 2 colunas (sidebar + conteúdo principal).
  • O menu de navegação deve ser horizontal no desktop e vertical/empilhado no mobile.
  • Teste redimensionando o navegador ou usando o modo de dispositivos móveis do DevTools (F12 → ícone de celular).

5. Imagens responsivas: técnicas profissionais

Imagens são frequentemente o maior problema em sites responsivos – podem ficar distorcidas, cortadas ou demorar muito para carregar em dispositivos móveis. Existem três técnicas principais para resolver isso.

5.1 Imagem fluida com CSS (a mais simples e essencial)

img {
    max-width: 100%;   /* imagem nunca ultrapassa o container */
    height: auto;      /* mantém a proporção original */
    display: block;    /* remove espaço extra abaixo da imagem */
}

Isso já resolve 80% dos casos: a imagem redimensiona proporcionalmente ao container.

5.2 Atributo srcset – imagens diferentes por resolução

O srcset permite fornecer múltiplas versões da mesma imagem, e o navegador escolhe a mais adequada baseado na largura da tela e densidade de pixels (ex: telas Retina).

<!-- Versão simples: imagens com larguras diferentes -->
<img src="imagem-pequena.jpg"
     srcset="imagem-media.jpg 800w,
             imagem-grande.jpg 1200w,
             imagem-hd.jpg 2000w"
     sizes="(max-width: 600px) 100vw, (max-width: 1200px) 80vw, 1200px"
     alt="Descrição da imagem">

5.3 Elemento <picture> – direção de arte (art direction)

Quando a imagem precisa ser visualmente diferente em cada dispositivo (não apenas maior/menor), use <picture>. Exemplo: em desktop você mostra uma foto larga com várias pessoas; em mobile, mostra um recorte focado no personagem principal.

<picture>
    <source media="(min-width: 1024px)" srcset="imagem-desktop.jpg">
    <source media="(min-width: 768px)" srcset="imagem-tablet.jpg">
    <source media="(max-width: 767px)" srcset="imagem-mobile.jpg">
    <img src="imagem-fallback.jpg" alt="Imagem responsiva com art direction">
</picture>
Exemplo de Art Direction: mesma imagem com diferentes cortes

5.4 Background responsivo

.hero {
    background-image: url('mobile-bg.jpg');
    background-size: cover;
    background-position: center;
    min-height: 300px;
}

@media (min-width: 768px) {
    .hero {
        background-image: url('tablet-bg.jpg');
        min-height: 400px;
    }
}

@media (min-width: 1024px) {
    .hero {
        background-image: url('desktop-bg.jpg');
        min-height: 500px;
    }
}

🖼️ Micro-atividade 5: Banner responsivo com picture e srcset

Crie uma página que exiba um banner hero no topo. Use <picture> para fornecer três versões da imagem:

  • Mobile (até 767px): imagem focada no centro, altura 300px.
  • Tablet (768px a 1023px): imagem mais larga, altura 400px.
  • Desktop (1024px+): imagem completa, altura 500px.
  • Além disso, configure um fallback com srcset para densidade de pixels (1x, 2x) – opcional mas recomendado.

🎯 PROJETO FINAL – Landing Page Responsiva para Agência Criativa

Desafio completo: Construa um site profissional para uma agência de design e desenvolvimento. Requisitos obrigatórios:

📄 HTML Semântico (peso 15%):
  • Use header, nav, main, section, article, aside, footer.
  • Títulos organizados hierarquicamente (h1, h2, h3).
  • Menu de navegação com links âncora para as seções da página.
🎨 Layout com Grid + Flexbox (peso 25%):
  • Layout principal da página com CSS Grid (use grid-template-areas).
  • Cards de serviços ou projetos com Flexbox (flex-wrap, gap, justify-content).
  • Menu de navegação horizontal em desktop, vertical em mobile (via media query).
📱 Responsividade com Media Queries (peso 20%):
  • Pelo menos 2 breakpoints (tablet e mobile).
  • Estratégia Mobile First (comece pelo CSS base para mobile).
  • Teste em diferentes resoluções usando o DevTools.
🖼️ Imagens Responsivas (peso 15%):
  • Banner hero com <picture> ou srcset (mínimo 2 versões).
  • Todas as imagens com max-width: 100%; height: auto;.
  • Atributo alt descritivo em todas as imagens.
✨ Efeitos e Interatividade (peso 15%):
  • Botões com transições (hover muda cor/background).
  • Cards com efeito de escala no hover (transform: scale(1.03)).
  • Links do menu com sublinhado animado ou destaque suave.
🎯 Paleta de Cores e Estilo (peso 10%):
  • Defina variáveis CSS para as cores principais.
  • Use uma paleta harmônica (ex: #0a1c3a, #2c7da0, #e9ecef, #f4a261).
  • Tipografia legível com contraste adequado.

Entrega: Um único arquivo HTML (com CSS interno ou externo) que represente uma landing page completa, funcional e responsiva. O projeto deve rodar em qualquer navegador moderno e se adaptar perfeitamente de 320px até 1920px de largura.
