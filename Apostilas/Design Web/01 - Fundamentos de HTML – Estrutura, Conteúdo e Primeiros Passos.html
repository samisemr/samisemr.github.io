1. O que é HTML e por que ele é a base da web?

HTML (HyperText Markup Language) não é uma linguagem de programação – é uma linguagem de marcação. Enquanto linguagens de programação (como JavaScript, Python, Java) executam lógica, cálculos e controlam comportamentos, o HTML tem uma única missão: estruturar e organizar o conteúdo de uma página. Pense na construção de uma casa: o HTML seria as paredes, o telhado e os cômodos (a estrutura bruta). O CSS seria a pintura, os revestimentos e a decoração. O JavaScript seria a fiação elétrica, os interruptores e os mecanismos que fazem as coisas acontecerem dinamicamente.

Por que todo profissional de multimídia e desenvolvimento precisa dominar HTML? Porque qualquer entrega digital – um portfólio, um sistema interativo, uma landing page de vendas, um aplicativo híbrido – utiliza HTML como esqueleto. Sem ele, o navegador não sabe onde colocar um texto, uma imagem ou um vídeo.

1.1 Breve história e evolução até HTML5

HTML foi criado por Tim Berners-Lee em 1991 enquanto trabalhava no CERN (laboratório europeu de física de partículas). A ideia era permitir que cientistas compartilhassem documentos de forma interligada através de hyperlinks. O primeiro documento HTML era absurdamente simples – cerca de 18 tags. Em 1995, surgiu o HTML 2.0, depois vieram versões 3.2, 4.0 (com suporte a tabelas e frames). O grande marco foi o HTML5, lançado oficialmente em 2014, que trouxe recursos nativos para multimídia: as tags <video> e <audio> sem precisar de plugins como Flash, além de elementos semânticos (<article>, <section>, <nav>) e APIs poderosas (canvas, geolocalização, armazenamento local). Hoje, todo navegador moderno suporta HTML5, então você não precisa se preocupar com versões antigas.

1.2 Arquitetura Cliente x Servidor: como o HTML chega na sua tela

🌐 Funcionamento prático:
- Você digita um endereço (ex: https://meusite.com) no navegador.
- O navegador (cliente) faz uma requisição HTTP para o servidor onde o site está hospedado.
- O servidor responde enviando um arquivo .html (e possivelmente CSS, JS, imagens).
- O navegador então parseia (lê e interpreta) o HTML, construindo a DOM (Document Object Model) – uma árvore de elementos. Em seguida, aplica os estilos e exibe a página.
- Importante: o HTML é estático; cada requisição recebe o mesmo código. Para conteúdo dinâmico, usamos linguagens de back-end (PHP, Node.js) que geram HTML sob demanda.

2. Ambiente de desenvolvimento: ferramentas práticas

Para escrever HTML, qualquer editor de texto serve (até o Bloco de Notas). Mas para produtividade, o Visual Studio Code (VS Code) é a escolha da indústria. Diferenciais: autocompletar tags, realce de sintaxe, emmet (atalhos como ! que gera estrutura HTML completa), e extensões que transformam o fluxo de trabalho.

Passo a passo para começar do zero:

  1. Baixe e instale o VS Code (gratuito).
  2. Crie uma pasta no computador (ex: meu-primeiro-site).
  3. No VS Code: Arquivo → Abrir Pasta → selecione a pasta criada.
  4. Clique no ícone "Novo Arquivo" ou pressione Ctrl + N. Salve como index.html (o nome index é o padrão que servidores procuram).
  5. Digite ! e tecle Enter – o VS Code expande para a estrutura completa do HTML5.
  6. Instale a extensão Live Server. Com o arquivo HTML aberto, clique com botão direito e escolha "Open with Live Server". A cada Ctrl+S, o navegador atualiza automaticamente.

3. Estrutura mínima de uma página HTML

<!DOCTYPE html>
<html lang="pt-BR">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Título da aba do navegador</title>
</head>
<body>
    <!-- Todo conteúdo visível para o usuário fica aqui -->
</body>
</html>

Explicação detalhada de cada parte:
- <!DOCTYPE html>: instrução obrigatória que diz ao navegador: "estamos usando HTML5". Sem isso, o navegador pode entrar em modo de compatibilidade (quirks mode) e renderizar a página de forma incorreta.
- <html lang="pt-BR">: elemento raiz. O atributo lang define o idioma principal – importante para leitores de tela e SEO.
- <head>: contém metadados invisíveis, como codificação de caracteres (meta charset="UTF-8" garante acentos e emojis), viewport (controle de escala em dispositivos móveis), título da aba, links para CSS e scripts.
- <body>: tudo que o usuário vê: textos, imagens, vídeos, botões, formulários.

4. Tags de texto: títulos, parágrafos, quebras e linhas horizontais

Títulos: de <h1> a <h6>
O <h1> representa o título principal da página – deve haver apenas um por página, por questões de SEO e acessibilidade. Os níveis seguintes (<h2> a <h6>) servem para subtítulos e seções hierárquicas. Não pule níveis (ex: não use <h4> depois de <h2> sem um <h3> no meio).

Parágrafos e quebras: <p> cria blocos de texto com espaçamento automático antes e depois. <br> é uma quebra de linha forçada (útil para endereços ou poemas). <hr> insere uma linha horizontal temática para separar seções.

<h2>Destinos mais inspiradores</h2>
<p>Conheça lugares que combinam tecnologia e arte.<br>Nova York, Tóquio e Berlim lideram o ranking.</p>
<hr>
<h3>Dicas práticas</h3>
<p>Leve sempre um caderno de esboços.</p>

5. Listas organizadas e não ordenadas

Listas são fundamentais para menus, requisitos de projeto, etapas de processo. <ul> cria lista com marcadores (bolinhas, quadrados). <ol> cria lista numerada automaticamente (1, 2, 3...). Cada item fica dentro de <li>.

6. Links e Imagens – Navegação e conteúdo visual

Links (âncoras): a espinha dorsal da web

O elemento <a> (âncora) conecta páginas, arquivos, seções da mesma página ou endereços externos. Atributo obrigatório: href (Hypertext Reference). target="_blank" abre o link em nova aba – útil para links externos, pois não tira o usuário do seu site. Links internos (para outras páginas do mesmo projeto) usam apenas o nome do arquivo: href="contato.html". Âncoras internas: href="#secao1" e no destino coloque id="secao1".

<a href="https://www.behance.net" target="_blank">Veja inspirações no Behance</a>
<a href="portfolio.html">Acesse meu portfólio</a>
<a href="#contato">Ir para formulário de contato</a>

Imagens: a tag <img>

A tag <img> é vazia (não tem fechamento). Atributos essenciais: src (caminho da imagem, pode ser local ou URL externa) e alt (texto alternativo descritivo – obrigatório para acessibilidade, lido por leitores de tela e exibido caso a imagem não carregue). Atributos opcionais: width e height (em pixels ou porcentagem). Para imagens de alta resolução, recomenda-se usar ferramentas de compressão (TinyPNG) e formatos modernos como WebP.

<img src="imagens/foto-perfil.jpg" alt="Retrato do designer multimídia trabalhando no tablet" width="300">

7. Tabelas: dados organizados em linhas e colunas

Tabelas não devem ser usadas para layout (isso é função do CSS Grid/Flexbox). Use tabelas apenas para dados tabulares: planilhas, horários, comparações de preços. Estrutura: <table> → linhas (<tr>) → células de cabeçalho (<th>) ou células comuns (<td>).

Dica de acessibilidade: use <thead>, <tbody> e <th scope="col"> para melhorar a interpretação por leitores de tela.

8. Formulários: capturando dados do usuário

Formulários são a principal forma de interação com o usuário: leads de contato, pesquisas, cadastros. A tag <form> agrupa os campos. Os atributos mais importantes: action (para onde os dados serão enviados) e method (GET – dados visíveis na URL, ou POST – dados enviados no corpo da requisição, mais seguro).

⚠️ Atenção: O HTML sozinho não processa os dados do formulário. Para receber as informações, você precisa de um back-end (PHP, Node, Python) ou usar serviços como Formspree, Netlify Forms ou Google Forms. A estrutura visual, no entanto, é 100% responsabilidade do HTML.

9. Boas práticas, semântica e próximos passos

Semântica HTML: usar as tags corretas para cada tipo de conteúdo. Ex: <nav> para menus, <article> para posts independentes, <aside> para barras laterais, <footer> para rodapé. Isso melhora SEO e acessibilidade.

Acessibilidade (a11y): sempre forneça alt descritivo em imagens; use <label> associado a inputs via for/id; evite apenas cores para transmitir informação; mantenha contraste legível.

Organização de arquivos: mantenha as imagens na pasta img/, CSS em css/, scripts em js/. Nunca use espaços ou caracteres especiais em nomes de arquivo – use hífen ou underline.

10. Resumo rápido para consulta

  • ✅ HTML = estrutura; CSS = estilo; JS = comportamento.
  • ✅ Toda página começa com <!DOCTYPE html>.
  • ✅ Tags de texto: <h1> a <h6>, <p>, <br>, <hr>.
  • ✅ Links: <a href="url" target="_blank">. Imagens: <img src="..." alt="...">.
  • ✅ Tabelas: para dados tabulares, sempre envolva em div class="table-wrapper" para rolagem horizontal em mobile.
  • ✅ Formulários: use <label>, inputs com type adequado, e lembre-se que o envio requer back-end.

Use apenas HTML (sem CSS ainda) – foque na estrutura semântica correta.
