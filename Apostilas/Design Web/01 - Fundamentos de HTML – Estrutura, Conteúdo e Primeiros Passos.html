Fundamentos de HTML – Estrutura, Conteúdo e Primeiros Passos

Esta apostila apresenta os conceitos essenciais de HTML (HyperText Markup Language), a base de toda página web. O conteúdo foi adaptado para alunos da área de Multimídia e também para quem está começando em programação web, com foco em objetividade, exemplos reais e aplicações diretas. Aqui você aprenderá desde a história até a criação de formulários, tabelas e links — sempre com situações que simulam projetos de mercado.

1. Conceitos iniciais: a base da web

HTML significa HyperText Markup Language (Linguagem de Marcação de Hipertexto). Diferente de linguagens de programação, o HTML estrutura o conteúdo: títulos, parágrafos, imagens, links. Pense no HTML como o esqueleto de uma página; o CSS é a “roupa” (estilos) e o JavaScript os “movimentos” (interações).

1.1 Por que HTML é essencial?

Qualquer site, aplicativo web, portfólio multimídia ou sistema interno usa HTML. Dominar as tags é o primeiro passo para atuar como front-end (parte visual) ou full-stack. Para um profissional de multimídia, saber HTML permite criar projetos interativos, animações contextualizadas e integrar vídeos/sons de forma semântica.

📌 Arquitetura Cliente x Servidor:
- Cliente: navegador (Chrome, Edge, Firefox) instalado no dispositivo do usuário.
- Servidor: computador remoto que armazena os arquivos do site.
- Quando você acessa um site, o navegador faz uma requisição HTTP ao servidor e baixa uma cópia do arquivo HTML, que é renderizada na tela.

1.2 Evolução rápida: HTML5

O HTML foi criado por Tim Berners-Lee em 1990. Atualmente, usamos o HTML5, que trouxe suporte nativo a vídeos, áudios, elementos semânticos (como <article>, <section>) e formulários mais poderosos. Não se preocupe com versões antigas: todo navegador moderno interpreta HTML5.

2. Ambiente de desenvolvimento: ferramentas práticas

Você só precisa de um editor de código. O mais recomendado hoje é o Visual Studio Code (VS Code), gratuito, multiplataforma e cheio de extensões que aceleram o trabalho. Veja o essencial:

  • Bloco de Notas: muito simples, mas sem recursos. Só para testes rápidos.
  • VS Code: editor poderoso com realce de sintaxe, Emmet (atalhos) e live preview.
  • Extensões úteis: Live Server (atualiza automática), Material Icon Theme (ícones) e Open in Browser.

Como começar no VS Code (passo a passo direto):

  1. Crie uma pasta no computador (ex: meu-site).
  2. Abra o VS Code → File → Open Folder → selecione a pasta.
  3. Clique no ícone "New File" (ou Ctrl+N), salve como index.html.
  4. Digite ! (exclamação) e pressione Enter – o VS Code gera a estrutura básica automaticamente.
  5. Instale a extensão Live Server. Depois, clique com botão direito no HTML e "Open with Live Server". Toda alteração salva atualiza o navegador na hora.

3. HTML na prática: estrutura e principais tags

3.1 Estrutura mínima de uma página

<!DOCTYPE html>
<html lang="pt-BR">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Minha primeira página</title>
</head>
<body>
    <!-- Todo conteúdo visível fica aqui -->
</body>
</html>

Explicação clara: <!DOCTYPE html> informa que é HTML5. <head> contém metadados (título da aba, codificação). <body> é o que o usuário enxerga.

3.2 Títulos, parágrafos, quebras e linhas horizontais

  • <h1> a <h6> – títulos em ordem de importância. Use apenas um <h1> por página (como título principal).
  • <p> – parágrafo. Adiciona espaçamento automático.
  • <br> – quebra de linha (não precisa fechar).
  • <hr> – linha horizontal para divisão de seções.
<h1>Bem-vindo ao Portfólio</h1>
<p>Este é um exemplo de parágrafo.<br>Texto com quebra de linha.</p>
<hr>
<h2>Projetos multimídia</h2>

4. Links, imagens e tabelas – construindo navegação e conteúdo rico

4.1 Links (âncoras) – a espinha da web

O elemento <a> conecta páginas, arquivos ou seções externas. Atributo obrigatório: href. Atributo target="_blank" abre em nova aba.

<a href="https://www.senai.br" target="_blank">Site do SENAI</a>
<a href="contato.html">Página de contato (mesma pasta)</a>
<a href="#galeria">Ir para seção de imagens</a>

4.2 Imagens – elementos visuais essenciais

Tag <img> (autofechada). Atributos obrigatórios: src (caminho da imagem) e alt (texto alternativo acessível). Dica: use width e height para controle de tamanho.

<img src="imagens/capa-projeto.jpg" alt="Preview do projeto de animação" width="600">

Importante para multimídia: otimize imagens (formatos WebP, JPG) para carregamento rápido. Sempre adicione alt descritivo – isso ajuda na acessibilidade e SEO.

4.3 Tabelas – dados organizados em linhas e colunas

Use tabelas para informações comparativas, grades ou dados técnicos. As tags principais: <table>, <tr> (linha), <th> (cabeçalho) e <td> (célula).

5. Formulários: capturando dados do usuário

Formulários são a porta de entrada para leads, cadastros e pesquisas. A tag <form> agrupa os campos. Ainda que o back-end não seja o foco agora, entender os tipos de campo é vital para criar interfaces interativas.

Exemplo de formulário simples (com HTML puro):

<form action="#" method="post">
    <label>Nome completo:</label><br>
    <input type="text" name="nome" placeholder="Seu nome"><br><br>
    
    <label>Qual serviço você procura?</label><br>

    <input type="radio" name="servico" value="design"> Design gráfico<br>
    
    <label>Ferramentas que domina:</label><br>
    <input type="checkbox" name="ferramentas" value="after"> After Effects<br>
    <input type="checkbox" name="ferramentas" value="premiere"> Premiere Pro<br>
    
    <label>Mensagem adicional:</label><br>

</form>

⚠️ Atenção: o HTML apenas exibe o formulário. Para processar os dados, você precisará de linguagem back-end (PHP, Node.js) ou serviços de form. Mas a estrutura visual e os campos são 100% HTML.

6. Boas práticas e próximos passos

Após dominar os elementos básicos (títulos, links, imagens, tabelas e formulários), você está apto a construir páginas web estáticas. Lembre-se:

  • Semântica: Use tags pelo significado, não apenas pela aparência. Ex: <strong> para ênfase forte, <em> para itálico semântico.
  • Organização de arquivos: Mantenha imagens em pastas separadas (ex: /img, /css).
  • Teste em diferentes navegadores – o HTML básico funciona em todos.
  • Acessibilidade: Sempre insira alt em imagens e use rótulos (<label>) em formulários.

  • Tags comuns: <h1>-<h6>, <p>, <a>, <img>, <table>, <form>.
  • Atributos importantes: href (links), src/alt (imagens), type (inputs).
  • Estrutura básica: DOCTYPE, html, head, body.
  • Ambiente profissional: VS Code + Live Server + atalho "!" para estrutura base.
