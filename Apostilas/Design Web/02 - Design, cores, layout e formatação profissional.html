1. O que é CSS e por que separar estilo do conteúdo?

CSS (Cascading Style Sheets) é a linguagem que define a apresentação visual de documentos HTML. Enquanto o HTML fornece a estrutura (títulos, parágrafos, imagens), o CSS controla cores, fontes, tamanhos, posicionamento, animações e responsividade. A separação entre conteúdo (HTML) e estilo (CSS) é uma das melhores práticas do desenvolvimento web porque:

  • Manutenção facilitada: Altere o visual de um site inteiro editando um único arquivo .css.
  • Reutilização: Um mesmo arquivo CSS pode estilizar dezenas ou centenas de páginas.
  • Carregamento mais rápido: O navegador baixa o CSS uma vez e o reutiliza.
  • Acessibilidade e SEO: Código limpo e semântico é melhor interpretado por leitores de tela e robôs de busca.

1.1 Sintaxe básica e anatomia de uma regra CSS

Toda regra CSS segue o mesmo padrão: seletor { propriedade: valor; }. O seletor aponta para o(s) elemento(s) HTML que você quer estilizar. As declarações ficam entre chaves, cada uma composta por uma propriedade (ex: color, font-size) e um valor (ex: blue, 16px).

/* Exemplo prático */
h1 {
    color: #0a1c3a;
    font-size: 32px;
    text-align: center;
}

1.2 Seletores essenciais (ID, classe, elemento, universal, agrupamento)

SeletorSintaxeExemploPrioridade (especificidade)
Elemento (tag)tag { }p { color: gray; }Baixa (1)
Classe.classe { }.destaque { background: yellow; }Média (10)
ID#id { }#cabecalho { font-size: 24px; }Alta (100)
Universal* { }* { margin: 0; padding: 0; }Mínima (0)
Agrupamentotag1, tag2 { }h1, h2, h3 { font-family: Arial; }Mesmo do seletor individual

Dica importante: ID é único por página (use para elementos que aparecem uma vez, como cabeçalho principal). Classes são reutilizáveis (perfeitas para botões, cards, alertas). Prefira classes sempre que possível – isso mantém seu CSS mais flexível.

<!-- HTML -->
<div id="menu-principal">
    <button class="botao botao-primario">Salvar</button>
    <button class="botao botao-secundario">Cancelar</button>
</div>

/* CSS */
.botao { padding: 10px 20px; border-radius: 8px; }
.botao-primario { background-color: #2c7da0; color: white; }
#menu-principal { background-color: #f8f9fa; }

<style>
    /* CSS interno - vale apenas para esta página */
    body { background-color: #f0f0f0; }
</style>

<h1 style="color: red;">Este título tem estilo inline (evite abusar)</h1>

1.4 Hierarquia (Cascata) – como o navegador resolve conflitos

O "Cascata" no nome CSS significa que as regras se sobrepõem seguindo uma ordem de prioridade. Quando há conflito, o navegador aplica:

  1. Origem e importância: Regras com !important têm prioridade máxima (use com moderação).
  2. Especificidade: ID > classe > elemento. Quanto mais específico, maior prioridade.
  3. Ordem de declaração: Se a especificidade é igual, a última regra declarada vence.
💡 Exemplo prático de conflito:
p { color: blue; } (especificidade 1)
.texto { color: red; } (especificidade 10)
→ O parágrafo com classe "texto" ficará vermelho, porque classe vence elemento.
→ Se houver p { color: green !important; }, o verde vence tudo.

2.2 Background (plano de fundo)

/* Fundo sólido */
body { background-color: #e9ecef; }

/* Fundo com imagem */
.hero {
    background-image: url('imagens/banner.jpg');
    background-size: cover; /* cobre todo o espaço */
    background-position: center;
    background-repeat: no-repeat;
}

/* Fundo gradiente (CSS3) */
.gradiente {
    background: linear-gradient(135deg, #2c7da0, #0a1c3a);
}

2.3 Bordas (border) e arredondamento

A propriedade border é um atalho que combina espessura, estilo e cor. border-radius arredonda os cantos.

.card {
    border: 2px solid #2c7da0;
    border-radius: 16px;
    /* bordas individuais: border-top, border-left, etc. */
}

2.4 Box Model – o coração do layout CSS

Todo elemento HTML é uma caixa retangular. O Box Model define como essa caixa ocupa espaço:

📦 Componentes do Box Model (de dentro para fora):
Content → onde fica o texto/imagem.
Padding → espaço interno entre o conteúdo e a borda. É transparente.
Border → contorno ao redor do padding.
Margin → espaço externo, separa um elemento dos outros.
⚠️ Cálculo de largura total padrão: width + padding-left + padding-right + border-left + border-right + margin-left + margin-right.
✅ Solução moderna: use box-sizing: border-box; – faz com que padding e borda sejam incluídos na largura definida. 
/* Sem border-box: largura real será 300px + 40px + 2px = 342px */
.caixa-antiga {
    width: 300px;
    padding: 20px;
    border: 1px solid black;
}

/* Com border-box: largura real continua 300px (padding e borda são "roubados" do conteúdo) */
.caixa-moderna {
    box-sizing: border-box;
    width: 300px;
    padding: 20px;
    border: 1px solid black;
}
Diagrama do Box Model: margin, border, padding, content

4. Float, clear e elementos block vs inline

float era amplamente usado para criar layouts em colunas, mas hoje foi superado por Flexbox e Grid. Ainda é útil para imagens flutuando dentro de textos (efeito de revista).

.imagem-flutuante {
    float: left;
    margin-right: 20px;
    width: 200px;
}
.clearfix::after {
    content: "";
    clear: both;
    display: table;
}

Elementos Block vs Inline: Block ocupa 100% da largura e quebra linha (<div>, <p>, <h1>). Inline ocupa só o espaço do conteúdo (<span>, <a>). Use display: inline-block para um híbrido.

5. Position, z-index e overflow

Valor
ComportamentoUso típico
staticPadrão, fluxo normal do documentoValor inicial, raramente definido
relativePosicionado relativamente à sua posição original. Desloca com top/left mas preserva o espaço original.Container para elementos absolute; pequenos ajustes
absolutePosicionado em relação ao ancestral mais próximo com position != static. Sai do fluxo (outros elementos ignoram seu espaço).Modais, tooltips, menus dropdown
fixedPosicionado em relação à viewport (janela do navegador). Não rola junto com a página.Cabeçalhos fixos, botões de voltar ao topo, banners de cookies
stickyHíbrido: se comporta como relative até rolar além do limite, depois se torna fixed.Menus que grudam no topo, cabeçalhos de seção
 
/* Botão flutuante de WhatsApp */
.whatsapp-btn {
    position: fixed;
    bottom: 20px;
    right: 20px;
    background-color: #25D366;
    border-radius: 50%;
    width: 60px;
    height: 60px;
    z-index: 1000;
}

/* Menu sticky */
.menu {
    position: sticky;
    top: 0;
    background: white;
    z-index: 100;
}

Z-index – controle de camadas

Elementos posicionados (relative, absolute, fixed, sticky) podem se sobrepor. O z-index define quem fica por cima: valores maiores sobem.

Overflow – lidando com conteúdo que transborda

.caixa-limitada {
    width: 300px;
    height: 150px;
    overflow: auto; /* adiciona barra de rolagem se necessário */
    /* overflow: hidden; corta o conteúdo */
    /* overflow: visible; padrão, vaza para fora */
}

Crie um botão "Contato" com position: fixed no canto inferior direito, fundo verde, texto branco, padding e borda arredondada. Adicione um elemento position: absolute dentro de um container relative para simular um tooltip que aparece ao passar o mouse. Use z-index para garantir que o tooltip fique sobre outros elementos.

5.1 Display: flex, grid, none e outras propriedades

display: none; remove completamente o elemento da tela (não ocupa espaço). visibility: hidden; esconde mas mantém o espaço ocupado. Flexbox e Grid são os métodos modernos de layout – esta apostila os aborda superficialmente, mas recomendamos aprofundamento.

6. Boas práticas e dicas de layout profissional

  • Use classes semânticas: .botao-primario em vez de .azul-grande – isso torna o código autoexplicativo.
  • Organização por componentes: Separe seu CSS em blocos lógicos (cabeçalho, formulários, cards, rodapé).
  • Mobile First: Escreva CSS para telas pequenas primeiro, depois use @media (min-width: 768px) para telas maiores.
  • Não use !important a menos que seja estritamente necessário – quebra a cascata natural e dificulta manutenção.
  • Utilize variáveis CSS (custom properties) para temas e valores reutilizáveis: :root { --cor-primaria: #2c7da0; } depois color: var(--cor-primaria);.
  • Teste no navegador com DevTools (F12) – você pode editar CSS ao vivo e ver o resultado imediatamente.
