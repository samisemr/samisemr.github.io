Folhas de Estilo em Cascata (CSS)
Design, cores, layout e formatação profissional

Se o HTML é o esqueleto da página, o CSS é a aparência: cores, fontes, espaçamentos, posicionamento e responsividade. Nesta apostila você aprenderá na prática como aplicar estilos, usar seletores, trabalhar com cores, bordas, margens, o modelo de caixa (box model) e técnicas de layout como float, display e position. Tudo com exemplos reais de mercado, direto ao ponto.

1. Introdução ao CSS: sintaxe e formas de uso

1.1 O que é CSS?

Cascading Style Sheets é a linguagem que define a apresentação visual de documentos HTML. Enquanto o HTML estrutura, o CSS estiliza. Separar conteúdo (HTML) de estilo (CSS) é uma prática essencial para manutenção e escalabilidade.

📌 Sintaxe básica:
seletor { propriedade: valor; }
Exemplo: h1 { color: #243E73; font-size: 32px; }
→ O seletor aponta para o elemento HTML. As declarações ficam entre chaves, cada uma com propriedade e valor, separadas por ponto e vírgula.

1.2 Seletores essenciais (ID, classe, universal e agrupamento)

  • Seletor de elemento: p { } → afeta todos os parágrafos.
  • Seletor de ID (#): #cabecalho { } → único na página, prioridade alta.
  • Seletor de classe (.): .destaque { } → reutilizável em vários elementos.
  • Seletor universal (*): * { margin:0; } → aplica a todos os elementos.
  • Agrupamento: h1, h2, p { text-align: center; } → evita repetição.
/* Exemplo prático */
#menu { background-color: #8BB4D9; }
.card-produto { border: 1px solid #A67E6F; padding: 15px; }
h1, h2 { font-family: 'Arial', sans-serif; }

1.3 Três formas de aplicar CSS

MétodoOnde escreverQuando usar
Externo (recomendado) Arquivo .css separado, linkado no <head> com <link> Sites com várias páginas, fácil manutenção
Interno Dentro da tag <style> no <head> Páginas únicas com estilo específico
Inline Atributo style diretamente na tag HTML Ajustes pontuais e urgentes (evite abusar) 
<!-- CSS externo (melhor prática) -->
<link rel="stylesheet" href="estilos.css">

<style> /* CSS interno */ body { background: #F2D8CE; } </style>

<h1 style="color: #243E73;">Inline pontual</h1>

1.4 Hierarquia (Cascata)

Quando há conflito, o navegador prioriza: 1º inline > 2º interno/externo (o último declarado) > 3º padrão do navegador. Além disso, seletores mais específicos (ID) ganham de classes.

🎨 Micro-atividade 1: Primeiros passos com seletores

Crie um arquivo HTML e um CSS externo. Estilize: um título <h1> com cor #243E73, centralizado; uma classe ".texto-destaque" com cor #A67E6F e negrito; e um ID "#rodape" com fundo #8BB4D9. Pratique o agrupamento para centralizar todos os títulos.

2. Cores, fundos, bordas, margin e padding

2.1 Cores no CSS

Você pode definir cores por: nomes (red, tomato), RGB (rgb(36,62,115)), hexadecimal (#243E73), HSL (hsl(220, 50%, 30%)). Use a ferramenta Adobe Color para criar paletas harmônicas.

2.2 Background (plano de fundo)

body { background-color: #F2D8CE; }
.caixa { background-image: url('fundo-textura.jpg'); background-repeat: no-repeat; background-size: cover; }

2.3 Bordas (border)

Propriedades: border-width, border-style (solid, dotted, dashed) e border-color. Atalho: border: 2px solid #A67E6F;. Bordas arredondadas: border-radius: 15px;.

2.4 Margin (margem externa) e Padding (preenchimento interno)

Margin: espaço fora da borda, afasta outros elementos. Padding: espaço dentro da borda, afasta o conteúdo da borda. Ambos podem ter valores individuais (top, right, bottom, left) ou abreviados.

.card {
    margin: 20px 15px;   /* 20px top/bottom, 15px left/right */
    padding: 30px;
    border: 1px solid #8BB4D9;
}
💡 Box Model essencial: Todo elemento é uma caixa: content → padding → border → margin. A largura total soma esses valores. Use box-sizing: border-box; para incluir padding e border na largura definida.

3. Largura, altura, formatação de texto e box model

3.1 Largura (width) e altura (height)

Podem ser definidas em pixels (px), porcentagem (%) ou viewport (vw/vh). Para layouts responsivos, use % ou flexbox/grid.

3.2 Formatação de texto e fonte

  • color: cor do texto.
  • text-align: left, center, right, justify.
  • text-decoration: underline, overline, line-through (evite sublinhar textos que não são links).
  • font-family: define a fonte. Prefira fontes seguras como Arial, Verdana, e sempre inclua fallback.
  • font-size: tamanho da fonte (px, em, rem).
  • font-weight: normal, bold, etc.
body { font-family: 'Arial', sans-serif; font-size: 16px; }
h1 { font-size: 2.5rem; text-align: center; letter-spacing: 1px; }

4. Float, clear e elementos block vs inline

4.1 Float e Clear

float faz o elemento “flutuar” à esquerda ou direita, permitindo que textos e outros elementos o envolvam. Muito usado em layouts antigos, hoje substituído por Flexbox/Grid, mas ainda útil para imagens com texto ao redor. clear força o próximo elemento a ficar abaixo do float.

.imagem-produto { float: left; margin-right: 20px; }
.clearfix { clear: both; }

4.2 Elementos block x inline

  • Block: ocupam 100% da largura, quebram linha (ex: <div>, <p>, <h1>).
  • Inline: ocupam apenas o espaço do conteúdo, ficam na mesma linha (ex: <span>, <a>, <img>).
  • Podemos alterar o comportamento com display: block; ou display: inline;.

Exemplo prático: menu horizontal com <li> (que são block) usando display: inline;.

5. Display, position, z-index e overflow

5.1 Display (além do block/inline)

display: none; → remove o elemento da tela (não ocupa espaço). display: flex; → modelo flexível (recomendado para layouts modernos).

5.2 Position (posicionamento avançado)

  • static: padrão, segue o fluxo da página.
  • relative: posiciona relativamente à sua posição original. Permite usar top, left, right, bottom.
  • absolute: posiciona em relação ao ancestral mais próximo com position diferente de static. Sai do fluxo normal.
  • fixed: fixo em relação à viewport (não rola junto).
  • sticky: híbrido entre relative e fixed.
.barra-fixa { position: fixed; bottom: 20px; right: 20px; background: #243E73; padding: 10px; }
.modal { position: absolute; top: 50%; left: 50%; transform: translate(-50%, -50%); }

5.3 Z-index – empilhamento de camadas

Controla a ordem de sobreposição. Só funciona em elementos posicionados (relative, absolute, fixed, sticky). Quanto maior o z-index, mais acima aparece.

5.4 Overflow – lidando com conteúdo extravasado

Define como lidar com conteúdo maior que o container: visible (padrão), hidden (corta), scroll (adiciona barra de rolagem), auto (rolagem automática).

.caixa-texto { width: 200px; height: 100px; overflow: auto; }

📐 Micro-atividade 3: Banner fixo com sobreposição

Simule um “botão flutuante” de atendimento: posicione um elemento position: fixed no canto inferior direito, com fundo #A67E6F, texto branco, padding e borda arredondada. Use z-index para garantir que fique sobre todo o conteúdo. Além disso, crie uma div com overflow: scroll e texto longo para testar a rolagem.

6. Bônus: boas práticas e dicas de layout

  • Organização: Use classes semânticas (ex: .botao-primario, .card-destaque).
  • Flexbox e Grid são hoje as principais ferramentas de layout (mais modernas que float). Recomendamos estudar em sequência.
  • Responsividade: Use unidades relativas (rem, %, vw) e media queries para adaptar o design a celulares.
  • Ferramentas do navegador: Utilize o “Inspecionar” (F12) para testar CSS ao vivo.
