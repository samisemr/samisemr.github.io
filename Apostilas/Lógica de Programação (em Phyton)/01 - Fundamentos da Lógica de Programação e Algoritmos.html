Fundamentos da Lógica de Programação e Algoritmos
O primeiro passo para se tornar um programador

Antes de escrever qualquer código, é preciso pensar de forma estruturada. A lógica de programação é a habilidade de organizar ideias e ações em uma sequência lógica para resolver problemas. O algoritmo é a materialização dessa sequência. Esta apostila apresenta os conceitos fundamentais: o que são algoritmos, como representá-los, como funciona um computador por dentro e o que são linguagens de programação. Tudo com exemplos práticos do dia a dia e exercícios para fixação.

1. Introdução à lógica e aos algoritmos

1.1 O que é lógica?

Lógica é a ciência que estuda as formas do pensamento e as operações intelectuais. No contexto da programação, lógica é a maneira como organizamos instruções e pensamentos para resolver um problema de forma racional. Todo dia usamos lógica: ao acordar, decidimos a ordem de nos vestir, tomar café e escovar os dentes. Essas decisões têm um motivo, mesmo que automático.

📌 Lógica na computação: “Maneira pela qual instruções, assertivas e pressupostos são organizados num algoritmo para viabilizar a implantação de um programa” (Dicionário Michaelis).

1.2 O que é um algoritmo?

Algoritmo é uma sequência finita de passos bem definidos e ordenados que, quando executados, resolvem um problema ou realizam uma tarefa. Algoritmos existem muito antes dos computadores: uma receita de bolo, as instruções para trocar um pneu, a fórmula de Bhaskara – tudo isso são algoritmos.

Exemplo clássico: fazer um sanduíche

  1. Pegue duas fatias de pão.
  2. Passe manteiga em uma das fatias.
  3. Coloque uma fatia de queijo sobre a manteiga.
  4. Coloque uma fatia de presunto sobre o queijo.
  5. Cubra com a segunda fatia de pão.
  6. Seu sanduíche está pronto!

Se você trocar a ordem (colocar o presunto antes do queijo), o resultado final pode ser diferente – mas ainda será um sanduíche. Já em operações matemáticas, a ordem é crítica.

Exemplo matemático: resolver [(a + b) × c + d]

  1. Calcule a + b (dentro dos parênteses).
  2. Multiplique o resultado por c.
  3. Some d ao resultado.

Qualquer outra ordem produziria um resultado incorreto, pois viola as regras da precedência de operadores.

2. Sistemas de computação – como um computador funciona

Algoritmos são executados por computadores. Para entender programação, é útil conhecer a arquitetura básica de um computador.

2.1 A máquina de von Neumann

Proposta em 1946 por John von Neumann, essa arquitetura é a base de praticamente todos os computadores modernos. Ela possui cinco componentes principais:

🖥️ [DIAGRAMA DA MÁQUINA DE VON NEUMANN]
Memória → ULA ←→ UC ←→ Entrada/Saída
  • Memória (RAM): armazena o programa em execução e os dados. Tudo em código binário (0 e 1).
  • Unidade Lógica e Aritmética (ULA): realiza cálculos (soma, subtração, comparações).
  • Unidade de Controle (UC): gerencia o fluxo de execução – decide qual instrução executar a seguir.
  • Dispositivos de entrada: teclado, mouse, scanner – por onde inserimos dados.
  • Dispositivos de saída: monitor, impressora – por onde o computador nos mostra resultados.

A ULA e a UC juntas formam a CPU (Central Processing Unit) – o "cérebro" do computador.

2.2 Bit, byte e palavra

  • Bit (binary digit): a menor unidade de informação. Pode ser 0 ou 1.
  • Byte: 8 bits. É a unidade básica de armazenamento (ex: um caractere 'A' ocupa 1 byte).
  • Palavra (word): bloco de bits que a CPU manipula de uma vez. Computadores atuais usam 64 bits (8 bytes).

2.3 Sistema operacional

O sistema operacional (SO) é um software que gerencia o hardware e permite que outros programas sejam executados. Ele abstrai as complexidades do hardware, permitindo que você escreva programas para o SO, não para uma máquina específica. Exemplos: Windows, Linux, macOS, Android, iOS.

3. Representações de algoritmos

Algoritmos podem ser representados de diferentes formas. Conhecer essas representações é essencial para planejar soluções antes de codificar.

3.1 Descrição narrativa

É a forma mais simples: escrever os passos em linguagem natural (português). É flexível, mas pode ser ambígua. Exemplo:

Algoritmo para verificar se dois números são iguais:
1. Ler dois valores (x e y).
2. Verificar se x e y são iguais.
3. Se forem iguais, mostrar "Valores iguais!".
4. Se forem diferentes, mostrar "Valores diferentes!".
5. Fim.

Problema: o passo 2 é vago – "como verificar"? A descrição narrativa é útil para rascunhos, mas não para implementação direta.

3.2 Pseudocódigo

O pseudocódigo é uma representação intermediária entre a linguagem natural e a linguagem de programação. Ele usa regras mais rígidas (palavras-chave como SE, ENTÃO, SENÃO, ENQUANTO) e é compreensível para humanos e fácil de converter para código real.

ALGORITMO verificar_igualdade
VAR
    x, y: INTEIRO
INÍCIO
    LER(x, y)
    SE x = y ENTÃO
        ESCREVER("Valores iguais!")
    SENÃO
        ESCREVER("Valores diferentes!")
    FIM_SE
FIM

3.3 Fluxograma

O fluxograma é uma representação gráfica do algoritmo, usando símbolos padronizados (ISO 5807). É excelente para visualizar o fluxo de decisões e repetições.

🔷 [SÍMBOLOS DE FLUXOGRAMA]
⚪ Terminal (início/fim) | ▭ Processo (ação/atribuição) | ◇ Decisão (condição) | ⬇️ Setas (fluxo)

Exemplo de fluxograma para verificar igualdade:

(Início) → [Ler x, y] → ◇ (x = y?) 
                              ├─ Sim → [Exibir "Iguais"] → (Fim)
                              └─ Não → [Exibir "Diferentes"] → (Fim)

✏️ Micro-atividade 3: Convertendo representações

Dado o algoritmo de cálculo da média de três números (somar os três e dividir por 3), represente-o de três maneiras:

  1. Descrição narrativa
  2. Pseudocódigo (use as palavras INÍCIO, LER, CALCULAR, ESCREVER, FIM)
  3. Fluxograma (desenhe no papel ou descreva os símbolos)

4. Linguagens de programação e compiladores

O computador só entende linguagem de máquina (binário: 0 e 1). Programar diretamente em binário é extremamente trabalhoso (imagine escrever "10110000 01100001" para somar dois números!).

4.1 O que é uma linguagem de programação?

Uma linguagem de programação é um conjunto de regras (sintaxe) que permite ao programador escrever instruções de forma mais próxima da linguagem humana. Exemplos: Python, Java, C, JavaScript.

4.2 Compilador vs. interpretador

  • Compilador: traduz o código-fonte inteiro para código de máquina antes da execução. Gera um arquivo executável (.exe, por exemplo). Exemplos: C, C++, Go.
  • Interpretador: traduz e executa o código linha por linha, em tempo real. Exemplos: Python, JavaScript, PHP.
🔄 Processo de compilação:
Código-fonte (texto) → Compilador → Código de máquina (binário) → Execução
Interpretação:
Código-fonte → Interpretador (executa linha a linha) → Resultado

Linguagens compiladas geralmente são mais rápidas; linguagens interpretadas são mais flexíveis e portáteis. O Python, que usaremos, é uma linguagem interpretada.

4.3 As linguagens mais populares (2023, segundo Stack Overflow)

📊 JavaScript (63%) | Python (49%) | Java (30%) | C# (27%) | C++ (22%) | PHP (18%)
  • JavaScript: líder em desenvolvimento web (front-end e back-end).
  • Python: versátil, fácil de aprender, líder em ciência de dados e IA.
  • Java: robusta, usada em sistemas corporativos e Android.
  • C/C++: alto desempenho, usadas em jogos, sistemas operacionais, embarcados.
  • C#: da Microsoft, forte em jogos (Unity) e aplicações Windows.
  • PHP: desenvolvimento web (ainda muito usado em back-end).

5. Linguagem Python – a escolha deste curso

Python é uma linguagem de propósito geral, interpretada, de alto nível e com sintaxe limpa e legível. Criada por Guido van Rossum no final dos anos 1980, seu nome é uma homenagem ao grupo britânico Monty Python (não à cobra!).

5.1 Por que Python é tão popular?

  • Sintaxe simples e intuitiva – parece quase português estruturado.
  • Multiparadigma: suporta programação estruturada, orientada a objetos e funcional.
  • Imensa coleção de bibliotecas – para web (Django, Flask), dados (Pandas, NumPy), IA (TensorFlow, PyTorch), jogos (Pygame) e muito mais.
  • Multiplataforma: roda em Windows, Linux, macOS, dispositivos móveis e até em microcontroladores.
  • Open source e com enorme comunidade ativa.

5.2 O Zen do Python

Tim Peters, um dos desenvolvedores da linguagem, escreveu um "manifesto" chamado Zen do Python. Alguns princípios:

  • Bonito é melhor que feio.
  • Explícito é melhor que implícito.
  • Simples é melhor que complexo.
  • Legibilidade faz diferença.
  • Deve haver um – e preferencialmente só um – modo óbvio para fazer algo.

5.3 Aplicações reais do Python

  • Google: muitos sistemas internos e o próprio buscador usam Python.
  • YouTube: grande parte da plataforma foi escrita em Python.
  • Netflix: usa Python para recomendações e análise de dados.
  • Industrial Light & Magic (Star Wars, Jurassic Park): softwares de efeitos visuais em Python.
  • Ciência de dados e IA: líder absoluto.
