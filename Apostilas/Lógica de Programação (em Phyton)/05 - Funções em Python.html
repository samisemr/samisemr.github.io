Funções em Python
Modularização, parâmetros, retorno, escopos e tratamento de exceções

Até agora, seus programas eram escritos de forma linear ou com loops, mas todo o código ficava no mesmo bloco. À medida que os programas crescem, essa abordagem se torna confusa e difícil de manter. As funções resolvem esse problema: elas permitem modularizar o código, reaproveitar trechos e organizar melhor a lógica. Nesta apostila, você aprenderá a criar suas próprias funções, trabalhar com parâmetros (obrigatórios e opcionais), retornar valores, entender escopo de variáveis (global vs. local) e até mesmo tratar erros com try/except.

1. Por que usar funções?

Funções são blocos de código reutilizáveis que executam uma tarefa específica. Em vez de repetir o mesmo código várias vezes, você escreve a função uma vez e a chama sempre que precisar.

🧭 Exemplo prático: Se você precisa exibir um menu estilizado com bordas em várias partes do programa, crie uma função mostrar_menu() e chame-a onde for necessário. Se precisar mudar o estilo da borda, altera apenas a função, não todas as ocorrências.

1.1 Vantagens das funções

  • Reutilização: escreva uma vez, use muitas vezes.
  • Organização: o código fica mais legível e estruturado.
  • Manutenção: alterações são feitas em um único lugar.
  • Testabilidade: é mais fácil testar partes isoladas do programa.

2. Definindo e chamando funções (sem parâmetros)

Em Python, usamos a palavra-chave def (definition) para definir uma função.

2.1 Sintaxe básica

def nome_da_funcao():
    # corpo da função (bloco indentado)
    print("Executando a função")

2.2 Exemplo: função que exibe uma borda

def realce():
    print("+" + "-" * 10 + "+")
    print("|" + " " * 10 + "|")
    print("|   MENU   |")
    print("|" + " " * 10 + "|")
    print("+" + "-" * 10 + "+")

# Chamando a função (programa principal)
realce()
print("1 - Opção 1")
print("2 - Opção 2")
realce()
+----------+
| |
| MENU |
| |
+----------+
1 - Opção 1
2 - Opção 2
+----------+
| |
| MENU |
| |
+----------+
📌 Importante: A função deve ser definida antes de ser chamada. O Python executa o código linha por linha; se você chamar a função antes de defini-la, ocorrerá um erro.

3. Funções com parâmetros

Parâmetros são dados de entrada que a função recebe para trabalhar. Eles tornam a função mais flexível.

3.1 Sintaxe com parâmetros

def saudacao(nome):
    print(f"Olá, {nome}! Seja bem-vindo(a).")

saudacao("Ana")   # Olá, Ana! Seja bem-vindo(a).
saudacao("João")  # Olá, João! Seja bem-vindo(a).

3.2 Múltiplos parâmetros

def soma(a, b):
    resultado = a + b
    print(f"{a} + {b} = {resultado}")

soma(5, 3)   # 5 + 3 = 8
soma(10, 20) # 10 + 20 = 30

3.3 Parâmetros opcionais (valor padrão)

Você pode definir valores padrão para parâmetros. Se o usuário não fornecer o argumento, o valor padrão é usado.

def potencia(base, expoente=2):
    return base ** expoente

print(potencia(3))     # 9 (3^2)
print(potencia(3, 3))  # 27 (3^3)
print(potencia(5, 4))  # 625 (5^4)

3.4 Ordem dos parâmetros

A ordem dos argumentos importa. O primeiro argumento vai para o primeiro parâmetro, e assim por diante.

def subtracao(a, b):
    return a - b

print(subtracao(10, 3))  # 7 (10 - 3)
print(subtracao(3, 10))  # -7 (3 - 10)

# Para inverter a ordem, use parâmetros nomeados
print(subtracao(b=10, a=3))  # -7 (3 - 10) - menos legível, evite

4. Retorno de valores (return)

Até agora, nossas funções apenas exibiam resultados. Mas muitas vezes queremos que a função retorne um valor para ser usado no programa principal. Para isso, usamos return.

4.1 Exemplo: função que retorna a soma

def somar(a, b):
    return a + b

resultado = somar(10, 5)
print(f"Resultado: {resultado}")  # Resultado: 15

# Pode usar diretamente
print(somar(7, 3))  # 10

4.2 Exemplo: função que retorna par ou ímpar

def par_ou_impar(numero):
    if numero % 2 == 0:
        return "par"
    else:
        return "ímpar"

n = int(input("Digite um número: "))
print(f"O número {n} é {par_ou_impar(n)}")

4.3 Múltiplos retornos

Uma função pode retornar mais de um valor (como uma tupla).

def operacoes(a, b):
    soma = a + b
    subtracao = a - b
    multiplicacao = a * b
    return soma, subtracao, multiplicacao

s, sub, m = operacoes(10, 3)
print(f"Soma: {s}, Subtração: {sub}, Multiplicação: {m}")
# Soma: 13, Subtração: 7, Multiplicação: 30

5. Escopo de variáveis (global x local)

Escopo é a região do programa onde uma variável é acessível. Python tem dois escopos principais:

  • Local: variáveis criadas dentro de uma função. Só existem dentro da função e são destruídas quando a função termina.
  • Global: variáveis criadas no programa principal (fora de qualquer função). Podem ser acessadas por qualquer parte do programa.

5.1 Exemplo: variável local

def teste():
    x = 10  # variável local
    print(f"Dentro da função: x = {x}")

teste()
# print(x)  # ERRO! x não existe fora da função

5.2 Exemplo: variável global

nome = "João"  # variável global

def saudacao():
    print(f"Olá, {nome}!")  # acessa a variável global

saudacao()  # Olá, João!

5.3 Modificando variáveis globais dentro da função (global)

Por padrão, se você tentar alterar uma variável global dentro de uma função, o Python cria uma variável local de mesmo nome. Para modificar a global, use a palavra-chave global.

contador = 0

def incrementar():
    global contador
    contador += 1

incrementar()
incrementar()
print(contador)  # 2
⚠️ Boa prática: Evite usar variáveis globais sempre que possível. Prefira passar valores como parâmetros e retornar resultados. Isso torna o código mais previsível e fácil de testar.

6. Tratamento de exceções (try/except)

Exceções são erros que ocorrem durante a execução do programa (ex: divisão por zero, conversão de tipo inválida). Com try/except, podemos tratar esses erros sem que o programa quebre.

6.1 Exemplo: tratando divisão por zero

try:
    num1 = int(input("Digite o primeiro número: "))
    num2 = int(input("Digite o segundo número: "))
    resultado = num1 / num2
    print(f"Resultado: {resultado}")
except ZeroDivisionError:
    print("Erro: divisão por zero não é permitida!")
except ValueError:
    print("Erro: você deve digitar números inteiros!")
except Exception as e:
    print(f"Ocorreu um erro inesperado: {e}")

6.2 Exemplo: validando entrada com try/except

def ler_inteiro(mensagem):
    while True:
        try:
            valor = int(input(mensagem))
            return valor
        except ValueError:
            print("Valor inválido! Digite um número inteiro.")

idade = ler_inteiro("Digite sua idade: ")
print(f"Idade: {idade}")

6.3 Finally (executa sempre)

try:
    arquivo = open("dados.txt", "r")
    conteudo = arquivo.read()
except FileNotFoundError:
    print("Arquivo não encontrado!")
finally:
    # Executa independentemente de erro ou sucesso
    print("Finalizando operação...")

7. Funções lambda (funções anônimas)

Lambda são funções pequenas e anônimas (sem nome) que podem ser escritas em uma única linha. São úteis para operações simples.

7.1 Sintaxe

lambda parametros: expressao

7.2 Exemplos

# Função que eleva ao quadrado
quadrado = lambda x: x ** 2
print(quadrado(5))  # 25

# Função com dois parâmetros
soma = lambda a, b: a + b
print(soma(3, 7))   # 10

# Usando lambda diretamente (sem variável)
print((lambda x: x * 2)(4))  # 8

# Útil com funções como map, filter
numeros = [1, 2, 3, 4, 5]
dobros = list(map(lambda x: x * 2, numeros))
print(dobros)  # [2, 4, 6, 8, 10]

8. Exercícios resolvidos

Exercício 1: Função com validação de string

def valida_string(pergunta, min_caracteres, max_caracteres):
    while True:
        texto = input(pergunta)
        if min_caracteres <= len(texto) <= max_caracteres:
            return texto
        print(f"O texto deve ter entre {min_caracteres} e {max_caracteres} caracteres.")

nome = valida_string("Digite seu nome (3 a 20 caracteres): ", 3, 20)
print(f"Nome válido: {nome}")

Exercício 2: Função que calcula fatorial

def fatorial(n):
    if n < 0:
        return None  # fatorial não definido para negativos
    resultado = 1
    for i in range(2, n + 1):
        resultado *= i
    return resultado

num = int(input("Digite um número para calcular o fatorial: "))
fat = fatorial(num)
if fat is None:
    print("Fatorial não definido para números negativos.")
else:
    print(f"{num}! = {fat}")

Exercício 3: Validação de número inteiro

def valida_int(pergunta, min_valor, max_valor):
    while True:
        try:
            valor = int(input(pergunta))
            if min_valor <= valor <= max_valor:
                return valor
            print(f"Valor deve estar entre {min_valor} e {max_valor}.")
        except ValueError:
            print("Digite um número inteiro válido.")

x = valida_int("Digite um valor entre 1 e 100: ", 1, 100)
print(f"Valor aceito: {x}")

Resumo da apostila

  • def: palavra-chave para definir funções.
  • Parâmetros: dados de entrada da função. Podem ser obrigatórios ou opcionais (valor padrão).
  • return: retorna um valor da função para quem a chamou. Sem return, a função retorna None.
  • Escopo local: variáveis criadas dentro da função. Não são acessíveis fora dela.
  • Escopo global: variáveis criadas fora das funções. Para modificar uma global dentro de uma função, use global.
  • try/except: tratamento de exceções (erros em tempo de execução).
  • Lambda: funções anônimas de uma linha, úteis para operações simples.
  • Reutilização: funções evitam repetição de código e facilitam a manutenção.
